# 電子報寄送

路徑：`/admin/newsletter/campaigns`（需具備 `newsletter` 權限）

電子報清單頁集中管理每一封電子報：從建立、編輯、測試寄送、正式群發，到發布為 RSS 給 Substack 抓取。素材的收集與 AI 產出請見〈[電子報素材](newsletter.md)〉。

---

## 電子報清單與狀態

清單以表格列出最近的電子報，欄位包含標題、主旨、狀態、成效、建立日期。

| 狀態 | 說明 |
|------|------|
| 草稿 | 尚未寄送，可持續編輯內容與收件名單 |
| 排程中 | 已排入寄送（藍色標示） |
| 已寄送 | 已完成寄送，並顯示寄出人數 |
| 部分失敗 | 部分收件人寄送失敗（紅色標示）；游標移到狀態上可看到失敗筆數與錯誤原因 |

已發布到 RSS 的電子報，狀態旁會另外顯示紫色「已發布」標記（與上述寄送狀態各自獨立）。

已寄送的電子報會顯示成效摘要：**寄達人數 · 開信數（開信率）· 點擊數**（來自 SendGrid 的開信 / 點擊追蹤）。

每一列可**開啟**進入編輯頁、**複製**成新草稿、就地**改名**，或**刪除**。

---

## 建立電子報

清單頁右上角提供四種建立方式：

| 方式 | 說明 |
|------|------|
| 素材 AI 撰寫 | 在〈[電子報素材](newsletter.md)〉的草稿頁按「AI 撰寫」，AI 依當月素材一次產出繁中 / English / 日本語三封電子報草稿 |
| Skill 匯入 | 上傳 Claude.ai newsletter-composer skill 產出的 zip（內含 `manifest.json` 與 `images/`），依內容含有的語言自動建立對應草稿 |
| AI 撰寫 | 直接在畫面輸入月份、開場白與每則故事大綱（可附圖片、連結），勾選「自動翻譯英文 + 日文版」後由 AI 產出，完成後跳至編輯頁 |
| 空白新增 | 建立一封完全空白的電子報，內容自行編輯 |

> 「素材 AI 撰寫」與「AI 撰寫」都會一次建立三種語言的電子報；「Skill 匯入」則依 zip 內實際包含的語言建立。

## 編輯內容

進入任一電子報即為編輯頁，可調整：

- **主旨**、**預覽文字（preview text）**
- **LINE / 群組宣傳短文**：另存一段社群宣傳文，可一鍵複製；按「三語批次匯入」可一次貼上繁中 / 英 / 日三語短文，自動存回同一批建立的三封電子報
- **內文**：提供四種檢視模式
  - **預覽**：以收件匣呈現的外觀檢視
  - **內嵌編輯**：直接在預覽上點選文字修改
  - **HTML**：編輯原始 HTML
  - **分割**：左側改 HTML、右側即時預覽
- **插入圖片**：於內嵌 / HTML / 分割模式，可上傳圖片並插入游標處

編輯頁另提供匯出與分享：

| 動作 | 說明 |
|------|------|
| 匯出 PDF | 以列印最佳化樣式輸出 PDF（自動隱藏退訂頁尾） |
| 匯出圖片 | 將整封電子報輸出成單張長圖（JPG），適合貼到 LINE / 社群 |
| 複製內文 | 複製去除頁首頁尾的內文（含格式），可直接貼進 Substack 編輯器 |
| Substack 連結 | 複製這封電子報的公開網頁版連結（需先發布到 RSS） |

> 修改後記得按「儲存草稿」。正式寄送前系統也會自動先儲存一次。

### 主旨 A/B 測試

在「主旨 B」欄填入第二個主旨即啟用 A/B 測試，並多出一個「**測試模式**」選項，可二選一：

| 測試模式 | 行為 |
|------|------|
| 小樣本＋自動送贏家（預設） | 先只寄給名單中一小部分作為測試組，比出開信率較高的主旨後，自動用贏家主旨寄給其餘所有人 |
| 全名單 50/50 對半 | 整份名單對半分成 A、B 兩組，各寄一個主旨，不挑選、也不自動補送贏家 |

選「**小樣本＋自動送贏家**」時另有兩個設定：

- **測試比例（%）**：先只寄給名單中這個比例的收件人作為測試組（可設 10–50%，預設 20%）；測試組內部再 A/B 各半，分別收到主旨 A 或主旨 B。
- **等待時間**：1 / 2 / 4 小時（預設 2 小時）。

等待時間結束後，系統（每 10 分鐘檢查一次）會比較測試組中兩個主旨的開信率，自動用**開信率較高**的主旨寄給名單其餘所有人（開信率相同時採用主旨 A）。這一段自動補寄若中途中斷（例如逾時），系統會在下次檢查時自動續寄，且已收過的人不會重複收信。

測試寄送一律使用主旨 A；留空「主旨 B」則不啟用 A/B 測試。各版本的開信率與獲勝主旨會顯示在成效分析頁。

> 選「**全名單 50/50 對半**」時不設測試組、不比開信率，全部寄完即結束。在 v7.8.0 之前建立、只填了主旨 B 但沒有測試比例的舊電子報，會自動套用此模式，行為與過去相同。

## 選擇收件名單

編輯頁右側可勾選一或多個收件名單。每個名單旁會顯示 **可寄送人數 / 總人數**：已退訂、退信、無效或寄送異常的訂閱者會被排除，只計入「可寄送」。

底部彙整所選名單的**可寄送總人數**，並標註因退信 / 退訂被排除的人數。名單的建立與維護請見〈[電子報素材 › 收件名單管理](newsletter.md#收件名單管理)〉。

## 測試寄送

在「測試寄送」欄輸入任一 email，按「測試寄送」即可只寄給該地址一封測試信。

測試寄送**不會**計入寄送紀錄、不影響電子報狀態、也不會寫入收件人成效。建議正式群發前先寄給自己確認排版。

---

## 正式寄送

按「正式寄送」即開始群發。系統行為：

- **寄送前二次確認**，並顯示實際可寄送人數
- **自動分批**：每批最多 1000 封，大型名單也能一次送完
- **重寄保護**：已完成寄送的電子報，不會因為再次點擊而整批重寄
- **不重複寄送**：系統會記錄每位已成功寄達的收件人；即使需要補寄，已收過的人也會自動略過，不會重複收信
- **格式錯誤的 email 會被略過**，並在結果中回報，方便回頭清理來源資料

寄送完成後，這封電子報會標為「已寄送」，並在清單頁顯示寄達 / 開信 / 點擊成效。

## 排程寄送

除了立即群發，也可以預約未來時間自動寄送：

- 在編輯頁選擇寄送的**日期與時間**，按「排程寄送」，電子報狀態即變為「排程中」（藍色標示）。
- 系統排程每 10 分鐘檢查一次，到達指定時間後約 10 分鐘內自動寄出，寄送行為與「正式寄送」完全相同。
- 尚未寄出前可**隨時取消排程**，電子報會退回「草稿」。

---

## 連結 UTM 標記

寄出的電子報內，所有指向網站的外部連結都會自動加上 UTM 參數（`utm_source=newsletter`、`utm_medium=email`、`utm_campaign=<電子報代稱>`），讓 Google Analytics 等分析工具能把網站流量歸因到這封電子報。連結若已自行帶有 `utm_*` 參數則保留不覆寫；退訂連結不會被加標記。

## 成效分析

已寄送（或部分失敗）的電子報，可在清單頁點該列的**圖表圖示**進入成效分析頁：

- **總覽數字**：寄達、失敗、開信（開信率）、點擊（點擊率）
- **A/B 主旨測試**：若啟用，顯示測試組人數、A、B 兩版的開信率，以及判定出的獲勝主旨（獲勝方標示徽章）；尚未判定時顯示「等待判定」倒數說明
- **失敗明細**：若有寄送失敗，列出每位失敗收件人的 email 與具體錯誤原因（最多 200 筆）
- **連結點擊**：每個連結的點擊數與不重複點擊人數
- **開信 / 點擊時間軸**：依時間分布的開信與點擊次數
- **其他事件**：退信、退訂、垃圾郵件檢舉的數量

部分失敗的電子報，頁面上方另會列出失敗筆數與各筆錯誤原因（可展開）。

## 互動分群名單

針對任一封已寄送的電子報，可依收件人的互動情況一鍵建立新的收件名單，方便後續再行銷（例如只補寄給未開信者）。在清單頁點該列的「建立分群名單」按鈕，選擇要建立的分群：

| 分群 | 收錄對象 |
|------|----------|
| 開信者 | 有開信紀錄的收件人 |
| 點擊者 | 有點擊紀錄的收件人 |
| 未開信 | 已寄達但未開信的收件人 |

建立後會產生一份新名單（名稱為「電子報標題 — 分群」），並自動跳至該名單頁。

## 退訂處理

每封電子報頁尾都會自動附上**退訂連結**（每位收件人專屬）。收件人點開後可以：

- **只取消特定名單**：頁面會列出他目前訂閱的名單，勾選想退出的名單即可，其餘名單照常收信。
- **取消所有訂閱**：登記全域退訂，往後所有電子報都不會再寄給他。

> 舊的退訂連結完全相容，行為不變（仍可直接全部退訂）。

寄送時系統會自動排除下列收件人，確保不寄給不該寄的人：

- 已退訂者
- 黑名單 email
- 聯絡人 email 狀態異常者（退信、無效等）
- 格式錯誤的 email

## 發布到 RSS（Substack）

編輯頁的「發布到 RSS」按鈕可將電子報加入公開 RSS feed（`/api/newsletter/feed.xml`）。

- Substack 會定期抓取此 feed，每一封新發布的電子報會成為 Substack 的一則**草稿貼文**
- feed 只保留**最新 20 封**已發布的電子報，並自動移除電子報的頁首、頁尾與退訂等 email 專用元件，只保留正文
- 發布後可再按「取消發布 RSS」將其移出 feed
- 發布後亦可用「Substack 連結」複製公開網頁版連結分享

## 電子報總覽儀表板

路徑：`/admin/newsletter/overview`（僅限 `super_admin`）

跨所有電子報的成效總覽，分三區：

- **寄送成效趨勢**：最近 12 次寄送的開信率與點擊率，可點進各封的成效分析頁。
- **名單健康度**：各收件名單的人數，以及 **180 天未開信**人數與比例——比例偏高代表該名單需要清理。
- **總計**：訂閱者、已退訂、黑名單的總數。
