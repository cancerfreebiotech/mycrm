# 電子報素材

路徑：`/admin/newsletter/draft/{YYYY-MM}`（需具備 `newsletter` 權限）

每期電子報由「素材草稿」組成。選定月份後分三個區塊編輯，最後用 AI 一鍵產出三語電子報。

---

## 三個區塊

| 區塊 | 說明 |
|------|------|
| 📌 本期重點 | 僅一則，出現在電子報最上方 |
| 📜 上月回顧 | 上個月已發生的活動 |
| 🔮 本月預告 | 本月即將發生的活動 |

---

## 新增素材

- **網頁新增**：點各區塊的「新增 Story」，填入標題、內文、活動日期、照片、連結
- **Telegram Bot**：把活動資訊傳給 Bot，快速建立素材

> ⚠️ **每則素材必須同時有「標題」和「內文」，才會被 AI 納入產出。**
> 只有標題的素材（例如從 Telegram 快速拍入、還沒補內文）會被略過。
> 按「AI 撰寫」後的預覽視窗會列出被略過的項目 —— 補上內文再重新生成即可。

---

## 調整順序（拖拉）

- **拖曳**卡片即可調整先後順序
- 可**跨區塊搬移**（上月回顧 ↔ 本月預告）
- 把某則**拖進「本期重點」**即設為當月焦點；本期重點只放一則，原本那則會自動退回上月回顧
- 支援觸控與鍵盤操作；放開即儲存

## 活動日期區間

- 可只填單一日期，或加填「結束日期（選填）」
- 有結束日期時，卡片與產出的電子報會顯示為區間（例如 `6/22 – 6/25`）

---

## 用 AI 產生電子報

1. 按右上角「**AI 撰寫**」
2. AI 依素材產出 **繁中 / English / 日本語** 三語預覽（主旨、內文 HTML、社群短文）
3. 檢視無誤後按「建立電子報草稿」→ 到〈[電子報寄送](newsletter-campaigns.md)〉編輯與寄送
4. 需要改動時可「重新生成」

> 產出順序依你在草稿頁排好的順序（拖拉的結果）。

## 匯出 JSON

「匯出 JSON」可將該期素材（原始繁中）匯出，供外部工具或 Claude.ai 電子報產生流程使用。

---

## 收件名單管理

路徑：`/admin/newsletter/lists`（可從〈[電子報寄送](newsletter-campaigns.md)〉清單頁右上「收件名單管理」進入）

收件名單是電子報的訂閱者群組，寄送時在此挑選對象。名單建立後即可在〈[電子報寄送](newsletter-campaigns.md)〉勾選使用。

### 從 CSV 匯入

- 名單頁右上「從 CSV 匯入」，選擇含兩欄「**名字**」+「**email**」的 CSV（email 必填；欄位順序不限，多餘欄位會忽略）
- 清單名稱由**檔名**自動產生，匯入後可改名
- 會略過：email 格式錯誤的列、同一份 CSV 內重複的 email
- ⚠️ **黑名單 / 已退訂的 email 仍會匯入名單**（依既定政策），只在匯入統計中另計「退信 / 退訂」數量；這些人會在**寄送時**才被自動排除
- 匯入的 email 若對應到現有聯絡人會自動連結；此流程**不會**建立新聯絡人

### 從聯絡人建立名單

- 在**聯絡人**頁篩選或勾選後，按「建立清單」即可把這批聯絡人建成新名單
- 與 CSV 匯入不同，這裡會**在建立時就排除**不可寄送者：黑名單、無 email、已退訂、退信 / 寄送異常
- 多個聯絡人共用同一 email 時，會自動合併為一位訂閱者

### 名單詳情與訂閱者狀態

點名單名稱進入詳情頁，可搜尋、篩選（狀態 / 國家 / 是否連結聯絡人）、排序，並逐筆管理訂閱者。上方統計卡：

| 統計 | 說明 |
|------|------|
| 總訂閱者 | 名單內全部人數 |
| 已連結聯絡人 | 已對應到 CRM 聯絡人的人數 |
| 可寄送 | 狀態正常、實際會收到信的人數 |
| 退信 / 無效 | 無法寄達的 email |
| 待處理 | 暫時失敗 / 信箱滿 / 寄件方或收件方擋信 |
| 已退訂 | 已退訂的人數 |

每位訂閱者會顯示健康狀態徽章：訂閱中、已退訂、退信、無效、暫時失敗、信箱滿、寄件擋、收件擋、垃圾檢舉。

其他操作：

- **新增**訂閱者：可「搜尋聯絡人」加入，或「直接輸入 Email」（已有訂閱者紀錄則直接加入，否則自動建立新訂閱者）
- **移除**單一訂閱者
- **同步 SendGrid**：從 SendGrid 拉回最新的退信 / 無效 / 退訂狀態，更新名單標記

### 匯出與統計

- 名單列表頁每列的**下載**圖示可將該名單訂閱者匯出為 CSV（含 email、姓名、公司、來源、加入時間、是否退訂；含 UTF-8 BOM，Excel 可正確開啟中日文）
- 編輯電子報時，各名單會即時計算「可寄送 / 總數」，方便估算實際觸及人數
