# 連絡先管理

パス：`/contacts`

---

## 連絡先一覧

### 検索と絞り込み

- **キーワード検索**：名前・メール・会社名を照合（あいまい検索）
- **タグ絞り込み**：タグボタンをクリックすると、そのタグを持つ連絡先のみを表示
- **デフォルト並び順**：「最終活動時刻」の降順（最近やり取りした連絡先が先頭に浮上 — 詳細は下記）
- **その他の並び順**：列ヘッダーをクリックして切替（氏名 / 会社 / 役職 / Email / タグ / 作成日時）
- **ページネーション**：1 ページあたり 20 件、下部にページナビゲーション

### エクスポート

「Excel エクスポート」または「CSV エクスポート」をクリックすると、現在の絞り込み結果をダウンロードできます（ページネーションの制限なし）。

## 連絡先の追加

パス：`/contacts/new`

4 つのセクションに分けて手動入力します：

| セクション | フィールド |
|-----------|-----------|
| **基本情報** | 名前（中国語/英語/現地語）、会社（中国語/英語/現地語）、役職 |
| **連絡先情報** | メール、第 2 メール、電話、第 2 電話、住所、ウェブサイト、**国** |
| **ソーシャルメディア** | LinkedIn、Facebook |
| **備考** | 備考テキスト |

### 名刺 AI 認識（複数枚）

1. 「名刺写真を選択」をクリック — 一度に最大 **6 枚**選択可能（表面・裏面など）
2. 「名刺をスキャン」をクリック — システムが圧縮・合成して AI 認識を実行
3. ページ左側に写真プレビュー、右側に AI 認識結果が並べて表示されます
4. 「フォームに適用」をクリックしてフィールドに入力するか、「無視」をクリックして手動入力

> 保存前にシステムがメールアドレスと名前の類似度を比較して重複連絡先を検出し、確認を促します。

---

## 一括アップロード

パス：`/contacts/batch-upload`

一度に最大 **50 枚**の名刺写真をアップロードできます：

1. クリックまたはドラッグで画像をアップロード（JPG / PNG 対応）
2. システムが最大 5 枚ずつ並行処理します（並行アップロード + AI 認識）
3. プログレスバーで進行状況を確認できます
4. 認識が完了するとプレビューテーブルが表示され、各エントリを個別に編集できます
5. 一括重複検出（既存の連絡先と照合）
6. 「すべて保存」をクリックして一括保存

## 連絡先の詳細

パス：`/contacts/[id]`

### 基本情報セクション

すべての連絡先フィールドを表示します。右上の「編集」ボタンをクリックして編集モーダルを開きます。

- **メール / 電話**フィールドの横にコピーアイコンがあり、クリックでクリップボードにコピーできます
- **国**フィールドには国旗 emoji + 名称が表示されます（例：🇯🇵 日本）

### 名刺管理

1 つの連絡先に複数の名刺を紐付けられます（表面・裏面・カスタムラベルなど）：
- 一度に最大 **6 枚**選択可能。AI が合成して認識し、差分のあるフィールドを確認用に表示します
- カードの右上の 🗑 をクリックして削除します

### インタラクションログのタイムライン

すべての記録を時系列の降順で表示します。タイプ：
- **メモ**（note）
- **会議**（meeting）
- **メール**（email）
- **システム**（system）— システムが自動生成

デフォルトで 20 件を読み込み、下部までスクロールすると自動でさらに読み込みます（無限スクロール）。

### メール送信

「メール送信」ボタンをクリックして高度なメール送信モーダルを開きます：
- **宛先（To / CC / BCC）**：編集可能、デフォルトは連絡先のメールアドレス
- **テンプレート**：選択すると件名・本文・添付ファイルが自動入力されます
- **AI 生成**：説明を入力すると、AI が受信者情報と最近のインタラクション記録を基に本文を自動生成（HTML、TipTap エディタ）
- **連絡先の写真**：bot `/p` や Web からアップロードした写真があれば、モーダルにサムネイルが表示されます。タップで添付、再タップで解除
- **一時添付ファイル**：今回の送信専用の添付ファイルをアップロード（1 ファイル最大 5MB）、テンプレートには保存されません

Microsoft Graph（Outlook）で送信。メールのインタラクション記録が自動作成され、該当連絡先は「最終活動」ソートで先頭に浮上します（次のセクション参照）。

## 最終活動時刻 (`last_activity_at`)

**目的**：最近やり取りした連絡先を一覧の先頭に自動的に浮上させ、名前で検索する手間を減らす。

### 計算ルール
`contacts.last_activity_at` は DB トリガーで自動更新され、以下のタイムスタンプの**最大値**を取る：
1. `interaction_logs.created_at` — type が `note` / `meeting` / `email` かつ **SendGrid 送信以外**
2. `contact_photos.created_at` — 任意の写真アップロード（bot `/p` または Web）
3. 該当なしなら `contacts.created_at` にフォールバック

### 動作マトリクス

| 操作 | 活動にカウント？ | インタラクション記録作成？ |
|---|---|---|
| 連絡先詳細ページからメール送信（Outlook/Graph） | ✅ | ✅ |
| Bot `/met` 訪問メモ | ✅ | ✅ (type=note または meeting) |
| Bot `/meet` 会議 | ✅ | ✅ (type=meeting) |
| Bot `/work` タスク | ✅ | ✅ (type=note) |
| Bot `/p` 写真アップロード | ✅ | ✅（メモ付きの場合 system 記録） |
| `/email/compose` Outlook 一括送信 | ✅ | ✅ |
| **`/email/compose` SendGrid 一括送信** | ❌ | ✅ |
| **Newsletter campaign（常に SendGrid）** | ❌ | ✅ |
| Bot `/a` 名刺追加（AI OCR） | ❌ (type=system) | ✅ |

### なぜ SendGrid はカウントされないのか
SendGrid は主に自動一括送信（ニュースレター、マーケティング）用途です。SendGrid メール送信が毎回 `last_activity_at` を更新すると、ニュースレターを 4000+ 購読者に送信した瞬間、全連絡先の「最終活動」が同じ時刻に塗り替えられ、ソートの意味が消失します。Outlook / Graph 送信は 1 対 1 または少人数への送信で、本当に追跡すべき「最近話した相手」の信号です。

インタラクション記録タイムラインには全送信履歴が残るため、送信内容の確認は問題ありません。ソート順に影響しないだけです。

### UI ヒント
`/email/compose` で SendGrid に切り替えると紫のバッジが表示されます：
> ⚠ SendGrid 送信は連絡先の最終活動時刻には加算されません（インタラクション記録は残ります）

## Hunter.io Email 自動検索

**目的**：新規連絡先に email が無い場合（OCR 取れず、フォーム未入力など）、システムが Hunter.io Email Finder を自動で 1 回照会。見つかれば connect.email へ書き込み、bot では通知、Web では静かに更新。

### 自動検索のトリガー
- Bot `/a` 名刺 OCR 完了、email 無し
- Bot `/li` LinkedIn OCR 完了、email 無し
- Bot `/p 氏名` 該当なし → 最小連絡先を作成（氏名のみ）→ 検索
- Web 一括アップロード、email 空欄の各行
- Web 手動追加 `/contacts/new`、email 欄未入力

### 毎日バッチ（cron）
- 毎日 **02:00 Asia/Taipei** に `/api/hunter/cron` が自動実行
- 1 回あたり最大 50 件（email 無し・30 日以内に未検索）
- 800 件のバックログは約 16 日で一巡
- Hunter アカウント残り検索数が 5 未満ならスキップ

### クレジット仕様
Hunter Free tier = 月 50 検索。ただし「該当なし」の検索は credit を消費しません。旧連絡先への大量スキャン（多くは見つからない）は安全。`/admin/hunter` で残り credits を確認可能。

### 管理ページ
`/admin/hunter` には以下が残ります：
- 統計（email 無し総数 / 未検索 / 検索済み未発見 / 今月検索済み）
- 手動バッチ実行（1 クリック最大 100 件）
- API キー設定
- `hunter_searched_at` リセット（全員を再検索したい場合）

## フィールドの説明

| フィールド | 説明 |
|-----------|------|
| `name` | 中国語名（主表示） |
| `name_en` | 英語名 |
| `name_local` | 現地語名（日本語など） |
| `company` | 会社（中国語） |
| `company_en` | 会社（英語） |
| `company_local` | 会社（現地語） |
| `title` | 役職 |
| `email` | 主メール |
| `second_email` | 第 2 メール |
| `phone` | 主電話 |
| `second_phone` | 第 2 電話 |
| `address` | 住所 |
| `website` | ウェブサイト |
| `linkedin_url` | LinkedIn |
| `facebook_url` | Facebook |
| `notes` | 備考 |
| `country_code` | 国コード（ISO 3166-1 alpha-2、例：`JP`）、countries テーブルに紐付け |
| `last_activity_at` | 最終活動タイムスタンプ（DB トリガーで自動更新、上記ルール参照） |
