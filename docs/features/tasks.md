# 任務管理

路徑：`/tasks`

任務管理支援兩種使用方式：**Web 介面**和 **Telegram Bot 指令**。

---

## 任務狀態

| 狀態 | 說明 |
|------|------|
| `pending` | 待處理（預設） |
| `done` | 已完成 |
| `postponed` | 已延後 |
| `cancelled` | 已取消 |

---

## Web 介面

### 三個分頁

| 分頁 | 顯示內容 |
|------|----------|
| **我的提醒** | 我建立的、且沒有指派對象的任務（自我提醒） |
| **我指派的** | 我建立的、有指派給其他人的任務 |
| **指派給我的** | 其他人指派給我的任務 |

### 搜尋

頂部搜尋框支援依任務標題關鍵字搜尋。

### 新增任務

點擊「+ 新增任務」按鈕，填寫：
- **任務標題**（必填）
- **描述**（選填）
- **截止時間**（選填）
- **指派給**：勾選一或多位成員（留空則為自我提醒）

### 操作

每筆任務右側有操作按鈕：

| 按鈕 | 動作 | 可用狀態 |
|------|------|---------|
| ✏️ | 編輯任務 | 所有狀態 |
| 🗑 | 刪除任務 | 所有狀態 |
| ✅ 完成 | 標記為 done | 僅 `pending` |
| ⏭ 延後 | 彈出視窗輸入新截止時間，狀態改為 postponed | 僅 `pending` |
| ❌ 取消 | 標記為 cancelled | 僅 `pending` |

> **助理代操作**：若主管設定了助理（見[個人設定](settings.md)），助理也可以替主管標記任務完成，`completed_by` 欄位會記錄實際操作者。

## Telegram Bot 建立任務

使用 `/work` 或 `/w` 指令，以**自然語言**描述任務：

```
/work 下週五前請陳小明整理 Q1 業績報告
/work 提醒自己明天早上寄提案給王大華
/w 請李四和王五本月底前完成客戶拜訪清單
```

Bot 會使用 Gemini AI 解析：
- **任務標題**
- **截止時間**（支援「明天」「下週五」「本月底」等）
- **負責人**（從成員名單中比對）

解析完成後：
1. 自動建立任務
2. 發 Telegram 通知給被指派的成員

---

## Telegram Bot 查看任務

使用 `/tasks` 或 `/t` 指令，列出你的待處理任務（我建立的 + 指派給我的）。

每筆任務附三個快速按鈕：
- **✅ 完成** — 立即標記完成
- **⏭ 延後** — 輸入新截止時間
- **❌ 取消** — 取消任務

---

## Teams Bot 通知

若已設定 [Teams Bot](../deployment/teams-setup.md)，新任務建立時會在 Teams 個人聊天送出 **Adaptive Card**，包含：
- 任務標題、截止時間
- **「標記完成」** 按鈕（直接在 Teams 點擊即可完成）
- **「前往任務管理」** 連結

---

## 到期提醒

系統每天 **09:00（Asia/Taipei）** 透過排程（Vercel Cron `task-reminders`）發送一則 Telegram「今日任務摘要」，內容包含：
- 🔴 **已逾期**的任務（`due_at` 已過且尚未完成）
- 🟡 **今日到期**的任務

摘要只會發給已綁定 Telegram 的成員；當天沒有逾期或到期任務的人不會收到任何訊息（空摘要即以「靜默」代替，無需退訂）。

每位成員收到的都是**自己的**任務，收件規則為：
- 被指派的成員（assignee）一律會收到
- 任務建立者僅在「自我提醒」時收到（即該任務沒有任何指派對象）；若建立者已把任務指派給其他人，則不會另外提醒建立者
