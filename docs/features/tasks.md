# 任務管理

路徑：`/tasks`

任務管理支援兩種使用方式：**Web 介面**和 **Telegram Bot 指令**。

---

## 任務狀態

| 狀態 | 說明 |
|------|------|
| `pending` | 待處理（預設） |
| `done` | 已完成 |
| `postponed` | 已延後 |
| `cancelled` | 已取消 |

---

## Web 介面

### 三個分頁

| 分頁 | 顯示內容 |
|------|----------|
| **我的提醒** | 我建立的、且沒有指派對象的任務（自我提醒） |
| **我指派的** | 我建立的、有指派給其他人的任務 |
| **指派給我的** | 其他人指派給我的任務 |

### 搜尋

頂部搜尋框支援依任務標題關鍵字搜尋。

### 新增任務

點擊「+ 新增任務」按鈕，填寫：
- **任務標題**（必填）
- **描述**（選填）
- **截止時間**（選填）
- **指派給**：勾選一或多位成員（留空則為自我提醒）

### 操作

每筆任務右側有操作按鈕（僅 `pending` 狀態可操作）：

| 按鈕 | 動作 |
|------|------|
| ✏️ | 編輯任務 |
| ✅ 完成 | 標記為 done |
| ⏭ 延後 | 彈出視窗輸入新截止時間，狀態改為 postponed |
| ❌ 取消 | 標記為 cancelled |

> **助理代操作**：若主管設定了助理（見[個人設定](settings.md)），助理也可以替主管標記任務完成，`completed_by` 欄位會記錄實際操作者。

## Telegram Bot 建立任務

使用 `/work` 或 `/w` 指令，以**自然語言**描述任務：

```
/work 下週五前請陳小明整理 Q1 業績報告
/work 提醒自己明天早上寄提案給王大華
/w 請李四和王五本月底前完成客戶拜訪清單
```

Bot 會使用 Gemini AI 解析：
- **任務標題**
- **截止時間**（支援「明天」「下週五」「本月底」等）
- **負責人**（從成員名單中比對）

解析完成後：
1. 自動建立任務
2. 發 Telegram 通知給被指派的成員

---

## Telegram Bot 查看任務

使用 `/tasks` 或 `/t` 指令，列出你的待處理任務（我建立的 + 指派給我的）。

每筆任務附三個快速按鈕：
- **✅ 完成** — 立即標記完成
- **⏭ 延後** — 輸入新截止時間
- **❌ 取消** — 取消任務

---

## Teams Bot 通知

若已設定 [Teams Bot](../bot/teams-setup.md)，新任務建立時會在 Teams 個人聊天送出 **Adaptive Card**，包含：
- 任務標題、截止時間
- **「標記完成」** 按鈕（直接在 Teams 點擊即可完成）
- **「前往任務管理」** 連結

---

## 到期提醒

系統每分鐘自動掃描到期任務（`due_at` 剛過去不超過 1 分鐘），發送 Telegram 提醒給：
- 所有被指派的成員
- 任務建立者（若不在指派名單中）
