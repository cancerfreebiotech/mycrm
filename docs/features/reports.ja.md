# レポート

パス：`/admin/reports`（super_admin のみ表示）

---

## レポートをすぐに生成

1. **開始日**と**終了日**を選択します
2. 「レポートを生成」をクリックします
3. ページ上でデータテーブルをプレビューします（新規連絡先 + インタラクション記録）
4. 「Excel をダウンロード」をクリックして `.xlsx` ファイルを取得します

### Excel の内容

| シート | 説明 |
|--------|------|
| **新規名刺** | 期間内に追加された連絡先（すべてのフィールドを含む） |
| **インタラクションログ** | 期間内のすべてのインタラクション記録（メモ/会議/メール） |

## Gmail 認可（自動送信）

定期スケジュールのレポートはバックエンドが Gmail 経由で自動送信するため、事前に一度だけ Google の認可を完了する必要があります。

**このページに「Gmail を連携」ボタンはありません。** 認可フローは super_admin がサインインした状態で、ブラウザで URL `/api/auth/gmail` を直接開いて開始します：

1. `/api/auth/gmail` を開きます（Google OAuth 認可画面へ自動的にリダイレクトされます）
2. Google の認可を完了します
3. システムが認可情報を保存します（有効期限切れ時に自動更新）

完了後、定期スケジュールはそのアカウントから自動送信できるようになります。この認可は通常一度だけ行えば十分です。

## 定期スケジュール

定期的なレポートの自動配信を設定します：

### スケジュールの追加

1. 「スケジュールを追加」をクリックします
2. 以下を入力します：
   - **スケジュール名**
   - **頻度**：毎週 / 毎月 / カスタム（Cron）— 「カスタム」を選択したときのみ Cron 式の欄が表示されます
   - **レポート期間（日数）**：各レポートが対象とする日数（1〜365）
   - **受信者（カンマ区切り）**：複数の受信者をカンマ区切りで入力できます
3. 保存後すぐにスケジュールが有効になります

> 編集：スケジュール行の ✏️ アイコンをクリックすると、既存の設定が入力された同じフォームが開きます。

### スケジュール一覧

| カラム | 説明 |
|--------|------|
| スケジュール名 | スケジュールの名前 |
| 頻度 | 実行頻度（毎週 / 毎月 / カスタム） |
| 受信者 | レポートの送信先 |
| ステータス | 有効 / 無効（クリックで切り替え） |
| 前回実行 | 直近の自動送信の時刻（台北時間）：成功は ✅、異常は ⚠️。未実行の場合は「未実行」と表示 |
| 操作 | ✏️ 編集、🗑 削除 |

### cron 式の例

| 頻度 | cron |
|------|------|
| 毎週月曜日午前 9 時 | `0 9 * * 1` |
| 毎月 1 日 | `0 9 1 * *` |
| 毎日午前 8 時 | `0 8 * * *` |

> 実行時間は UTC 基準です。日本時間は UTC+9 ですので、ご注意ください。
