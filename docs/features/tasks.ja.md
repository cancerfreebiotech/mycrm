# タスク管理

パス：`/tasks`

タスク管理は **Web 画面**と **Telegram Bot コマンド**の 2 種類の方法に対応しています。

---

## タスクのステータス

| ステータス | 説明 |
|-----------|------|
| `pending` | 未処理（デフォルト） |
| `done` | 完了 |
| `postponed` | 延期済み |
| `cancelled` | キャンセル済み |

---

## Web 画面

### 3 つのタブ

| タブ | 表示内容 |
|------|---------|
| **自分のリマインダー** | 自分が作成した、担当者なしのタスク（自己リマインダー） |
| **自分が割り当てたもの** | 自分が作成した、他のメンバーに割り当てたタスク |
| **自分に割り当てられたもの** | 他のメンバーから自分に割り当てられたタスク |

### 検索

上部の検索ボックスでタスクのタイトルのキーワード検索に対応しています。

### タスクの追加

「+ タスクを追加」をクリックして以下を入力します：
- **タスクタイトル**（必須）
- **説明**（任意）
- **期限**（任意）
- **割り当て先**：1 名以上のメンバーにチェックを入れます（空欄の場合は自己リマインダー）

### 操作

各タスクの右側に操作ボタンが表示されます（`pending` ステータスのみ操作可能）：

| ボタン | アクション |
|--------|-----------|
| ✏️ | タスクを編集 |
| ✅ 完了 | done としてマーク |
| ⏭ 延期 | ポップアップで新しい期限を入力、ステータスが postponed に変わる |
| ❌ キャンセル | cancelled としてマーク |

> **アシスタントによる代理操作**：マネージャーがアシスタントを設定している場合（[個人設定](settings.md) を参照）、アシスタントもマネージャーのタスクを完了にマークできます。`completed_by` フィールドに実際の操作者が記録されます。

---

## Telegram Bot でのタスク作成

`/work` または `/w` コマンドで**自然言語**を使ってタスクを説明します：

```
/work 来週金曜日までに山田さんに Q1 業績レポートをまとめてもらう
/work 明日の朝に田中さんに提案書を送ることを自分にリマインド
/w 鈴木さんと佐藤さんに今月末までに顧客訪問リストを完成させてもらう
```

Bot が Gemini AI を使って解析します：
- **タスクタイトル**
- **期限**（「明日」「来週金曜日」「今月末」などに対応）
- **担当者**（メンバー一覧から照合）

解析後：
1. タスクが自動作成されます
2. 割り当てられたメンバーに Telegram 通知が送信されます

---

## Telegram Bot でのタスク確認

`/tasks` または `/t` コマンドで未処理タスクの一覧を表示します（自分が作成 + 自分に割り当てられたもの）。

各タスクに 3 つのクイックボタンが付きます：
- **✅ 完了** — すぐに完了としてマーク
- **⏭ 延期** — 新しい期限を入力
- **❌ キャンセル** — タスクをキャンセル

---

## Teams Bot 通知

[Teams Bot](../bot/teams-setup.md) が設定されている場合、新しいタスクが作成されると Teams の個人チャットに **Adaptive Card** が送信されます。内容：
- タスクのタイトルと期限
- **「完了にする」**ボタン（Teams で直接クリックして完了可能）
- **「タスク管理へ移動」**リンク

---

## 期限リマインダー

システムが毎分自動的に期限切れのタスク（`due_at` が 1 分以内に過ぎたもの）をスキャンし、以下に Telegram リマインダーを送信します：
- 割り当てられたすべてのメンバー
- タスクの作成者（担当者リストにいない場合）
