# ニュースレター素材

パス：`/admin/newsletter/draft/{YYYY-MM}`（`newsletter` 権限が必要）

各号は「素材下書き」で構成されます。月を選び、3つのセクションで編集し、AI でワンクリックで三言語のニュースレターを生成します。

---

## 3つのセクション

| セクション | 説明 |
|-----------|------|
| 📌 今号のハイライト | 1件のみ、最上部に表示 |
| 📜 先月の振り返り | 先月のイベント |
| 🔮 今月の予告 | 今月の予定 |

---

## 素材の追加

- **ウェブ**：各セクションの「Story を追加」からタイトル・本文・日付・写真・リンクを入力
- **Telegram Bot**：イベント情報を Bot に送って素早く作成

> ⚠️ **各素材はタイトルと本文の両方がないと AI 生成に含まれません。**
> タイトルのみ（Telegram 取り込みで本文未記入など）はスキップされます。
> 「AI 生成」後のプレビューに除外項目が表示されるので、本文を追加して再生成してください。

---

## 並べ替え（ドラッグ＆ドロップ）

- カードを**ドラッグ**して順序を変更
- **セクション間の移動**も可能（先月 ↔ 今月）
- 「今号のハイライト」にドラッグすると注目記事に設定（ハイライトは1件、元の記事は先月へ戻ります）
- タッチ・キーボード対応。ドロップで保存

## イベント日の期間

- 単日、または「終了日（任意）」を追加
- 終了日がある場合、カードと生成されたニュースレターに期間表示（例：`6/22 – 6/25`）

---

## ニュースレターの生成

1. 右上の「**AI 生成**」をクリック
2. AI が **繁体字中国語／英語／日本語** のプレビュー（件名・本文 HTML・SNS 用文）を生成
3. 確認後「ニュースレター下書きを作成」→〈[ニュースレター配信](newsletter-campaigns.md)〉で編集・配信
4. 「再生成」でやり直し可能

> 記事の順序は下書きボードで並べた順（ドラッグ結果）に従います。

## JSON エクスポート

「JSON エクスポート」で当月の素材（繁体字中国語の原文）を書き出し、外部ツールや Claude.ai のニュースレター生成に利用できます。

## 受信者リスト管理

パス：`/admin/newsletter/lists`（〈[ニュースレター配信](newsletter-campaigns.md)〉一覧ページ右上の「宛先リスト管理」から移動できます）

受信者リストはニュースレターの購読者グループで、配信時にここから宛先を選びます。作成したリストは〈[ニュースレター配信](newsletter-campaigns.md)〉で選択できます。

### CSV からインポート

- リストページ右上の「CSVからインポート」で、2 列「**名字**」+「**email**」を含む CSV を選択します（email は必須。列の順序は問わず、余分な列は無視されます）
- リスト名は**ファイル名**から自動生成され、インポート後に変更できます
- スキップ対象：email の形式が不正な行、同じ CSV 内で重複した email
- ⚠️ **ブラックリスト / 配信停止の email もリストにインポートされます**（既定のポリシーによる）。インポート統計で「バウンス / 配信停止」として別途カウントされるだけで、**配信時**に自動的に除外されます
- インポートした email が既存の連絡先と一致する場合は自動的に紐付けられます。このフローで新しい連絡先が作成されることは**ありません**

### 連絡先からリストを作成

- **連絡先**ページで絞り込みまたは選択し、「リストを作成」を押すと、その集合から新しいリストを作成できます
- CSV インポートと異なり、ここでは**作成時に送信不可の人を除外**します：ブラックリスト、email なし、配信停止、バウンス / 送信異常
- 複数の連絡先が同じ email を共有している場合は、1 人の購読者にまとめられます

### リスト詳細と購読者ステータス

リスト名をクリックすると詳細ページが開き、検索・絞り込み（ステータス / 国 / 連絡先との紐付けの有無）・並べ替えができ、購読者を 1 件ずつ管理できます。上部の統計カード：

| 統計 | 説明 |
|------|------|
| 総購読者数 | リスト内の全員 |
| 連絡先と紐付け済み | CRM 連絡先に対応している人数 |
| 送信可能 | ステータスが正常で、実際にメールを受け取る人数 |
| バウンス / 無効 | 配信できない email |
| 保留中 | 一時失敗 / メールボックス満杯 / 送信側または受信側ブロック |
| 配信停止済み | 配信停止した人数 |

各購読者には健康ステータスのバッジが表示されます：購読中、配信停止済み、バウンス、無効、一時失敗、メールボックス満杯、送信側ブロック、受信側ブロック、迷惑メール報告。

その他の操作：

- 購読者の**追加**：「連絡先を検索」して追加、または「メールを直接入力」（購読者レコードが既にあればそのまま追加、なければ新しい購読者を自動作成）
- 個別の購読者の**削除**
- **SendGrid を同期**：SendGrid から最新のバウンス / 無効 / 配信停止ステータスを取得し、リストのマークを更新

### エクスポートと統計

- リストページの各行の**ダウンロード**アイコンで、そのリストの購読者を CSV エクスポートできます（email、氏名、会社、ソース、追加日、配信停止の有無を含む。UTF-8 BOM 付きで、Excel が中国語 / 日本語を正しく開けます）
- ニュースレターの編集時、各リストに「送信可能 / 総数」がリアルタイムで表示され、実際のリーチの見積もりに役立ちます
