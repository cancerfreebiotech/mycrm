# 初回ログイン

## ログイン方法

myCRM は **Microsoft アカウント（Azure AD）シングルサインオン**を使用しており、`@cancerfree.io` ドメインのアカウントのみ利用できます。

1. [https://mycrm-vert.vercel.app](https://mycrm-vert.vercel.app) にアクセスします
2. 「Microsoft アカウントでサインイン」をクリックします
3. 会社のメールアドレスとパスワードを入力します
4. アプリケーションの認可に同意します
5. 初回ログイン後、システムは **MFA（二段階認証）の設定** を求めます（下記参照）

> 初回ログイン後、システムは自動的に `users` テーブルにアカウントを作成します（ロールのデフォルトは `member`）。

---

## MFA（二段階認証）の設定

アカウントのセキュリティのため、**すべてのユーザーは MFA（二段階認証）の設定が必須**です。初回ログイン後、システムは自動的に MFA 設定ページに遷移し、設定を完了するまでどの機能も利用できません。

1. スマートフォンに認証アプリ（Google Authenticator、Microsoft Authenticator、1Password など）をインストールします
2. 画面に表示された QR コードをアプリでスキャンします（またはその下に表示される秘密鍵を手動で入力します）
3. アプリが生成する **6 桁の認証コード**を入力して登録を完了します
4. 登録が完了すると、システムを通常どおり利用できます

> 以降のログインごとに、認証アプリの現在の 6 桁コードの入力が求められます。

- 認証アプリを変更・再登録する場合：右上角 → **個人設定** → MFA セクションで管理します。
- スマートフォンの紛失などでコードを取得できない場合：super_admin に連絡し、**ユーザー管理**ページで MFA をリセットしてもらってから、再度ログインして登録し直してください。

## Telegram の連携

Telegram Bot 機能（名刺スキャン・タスク管理）を使用するには、先に Telegram アカウントを連携する必要があります：

1. Telegram で **`@userinfobot`** を検索し、任意のメッセージを送信して Telegram ID（数字のみ、例：`123456789`）を取得します
2. myCRM にログイン → 右上角 → **個人設定** → Telegram ID を入力して保存します
3. Telegram で **`@CF_CRMBot`**（CancerFreeCRM）を検索して会話を開始します

> 連携後、`/help` を送信すると利用可能なコマンド一覧が表示されます。

---

## Microsoft Teams Bot の連携

タスク通知を受け取り、Teams で直接タスクを完了にするには、一度連携操作を行う必要があります：

1. Microsoft Teams で Bot 名を検索します（管理者から提供）
2. **1 対 1 チャット**を開きます
3. 任意のメッセージを送信します（例：`help`）
4. Bot が自動的に AAD アカウントを解析し、CRM アカウントと紐付けます

連携が成功すると Bot が次のように返信します：
```
📋 myCRM Bot（連携済み：your.email@cancerfree.io）
タスク通知はここに自動送信されます。
```

> 連携は一度行うだけで済みます。以降、タスクが割り当てられると自動的に Teams 通知カードが届きます。

## 言語設定

myCRM は 3 つの言語に対応しています：**繁体字中国語**・**English**・**日本語**

切り替え方法（どちらでも可）：
- ヘッダー右上の 🌐 アイコンをクリック → 言語を選択
- **個人設定** → 言語選択セクション

言語設定はアカウントに保存され、次回ログイン時に自動的に適用されます。

## 画面の説明

ログイン後、左側にサイドバーが表示されます。上部はすべてのメンバーが使用できるコア機能、区切り線より下は権限付与が必要な高度な機能です。各ページの右下には AI アシスタントのフローティングボタンがあります。

**すべてのメンバーに表示（コア機能）：**

| 項目 | 説明 |
|------|------|
| ダッシュボード | ダッシュボードと統計 |
| 連絡先 | 名刺連絡先一覧 |
| 確認待ち名刺 | AI 認識後、確認待ちの名刺 |
| 認識失敗 | 自分がスキャンして認識に失敗した名刺 |
| アルバム | 名刺・イベント写真 |
| メモ検索 | 連絡先メモの検索 |
| タスク | タスク一覧 |
| レポート | 統計レポート |
| メールキャンペーン | Email マーケティング |
| 個人設定 | プロフィール・言語・MFA・Telegram 連携 |
| 使用説明書 | 本ドキュメント |
| 💬 フィードバック | フィードバック送信 |

> モバイルではハンバーガーメニュー、デスクトップではマウスオーバーで展開し、左下の矢印で折りたたみを固定できます。

**高度な機能（権限が必要）：**

区切り線より下の項目（タグ管理、未分類メモ、メールテンプレート、プロンプト管理、国管理、ニュースレター、スキャン失敗、重複連絡先、共用 Email、Email リカバリー、CamCard 取込、ゴミ箱、ユーザー管理）は、super_admin が**ユーザー管理**で機能ごとに権限を付与した後に利用できます。

**super_admin 専用：**

組織設定、AI モデル管理、システム健全性、MCP Tokens、MCP 活動、監査ログ、配信停止管理、DSAR照会、ニュースレター概要、フィードバック管理は super_admin のみに表示されます。

## フィードバックの送信

サイドバーの **💬 フィードバック**（`/feedback`）から、使用中の問題や欲しい機能をいつでもチームに報告できます：

1. 種類を選択：**機能リクエスト** または **バグ報告**
2. タイトルと詳細説明を入力（いずれも必須）
3. 必要に応じてスクリーンショットを 1 枚添付（バグ報告では推奨）
4. 送信後、管理者が管理画面の**フィードバック管理**で追跡・対応します

> 送信が完了すると確認画面が表示され、そのまま戻れます。

## ロールの説明

myCRM のロールは 2 種類のみです：

| ロール | 説明 |
|--------|------|
| `member` | 一般ユーザー。すべてのコア機能（ダッシュボード・連絡先・確認待ち名刺・認識失敗・アルバム・メモ検索・タスク・レポート・メールキャンペーン・個人設定・使用説明書・フィードバック・AI アシスタント）を使用できます。高度な機能には別途権限付与が必要です。 |
| `super_admin` | システム管理者。すべての機能を備え、ユーザー管理、機能権限の付与、他ユーザーの MFA リセット、AI モデルやシステム設定の管理ができます。 |

高度な機能は**機能ごとの権限付与（granted features）**で制御されます：super_admin が**ユーザー管理**ページで個々のメンバーに必要な機能（例：ニュースレター、タグ管理、ユーザー管理など）を有効化すると、そのメンバーはロールを変更せずにその機能を使用できます。
