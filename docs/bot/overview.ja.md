# Telegram Bot 概要

myCRM Telegram Bot はシステムの主要なデータ入力チャンネルです。以下の機能に対応しています：

| 機能 | コマンド |
|------|---------|
| 名刺スキャン（写真アップロード） | 写真を直接送信 |
| 連絡先検索 | `/search` |
| インタラクションメモの追加 | `/note` |
| メール送信 | `/email` |
| 名刺裏面の追加 | `/add_back` |
| タスク作成 | `/work` |
| 自分のタスクを確認 | `/tasks` |
| 組織メンバー一覧 | `/user` |
| ヘルプ | `/help` |

---

## 利用フロー（名刺スキャン）

```
ユーザーが名刺写真を送信
        ↓
Bot が画像を Supabase Storage にアップロード
        ↓
Gemini AI を呼び出してすべてのフィールドを認識
        ↓
Bot が認識結果を表示し、保存するか確認
        ↓
ユーザーが選択：
  ✅ 保存 → contacts テーブルに書き込み
  ❌ 破棄 → 画像を削除
```

---

## マルチステップ会話

Bot は `bot_sessions` テーブルを使って、複数ステップの会話状態を管理します（例：連絡先検索 → 選択 → メモ内容入力）。各ユーザーの会話状態は独立して管理されます。

---

## 連続写真送信の保護

同一ユーザーに未確認の名刺が **5 枚**を超えて溜まっている場合、Bot は新しい写真を拒否し、既存の未確認項目を先に処理するよう促します。
