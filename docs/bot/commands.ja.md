# Bot コマンド一覧

| コマンド | 省略形 | 説明 |
|---------|--------|------|
| `/help` | `/h` | 利用可能なすべてのコマンドを表示 |
| `/lang [zh\|en\|ja]` | — | Bot の返信言語を切り替え（繁体字中国語／英語／日本語） |
| `/search [キーワード]` | `/s` | 連絡先を検索（名前/Email のあいまい検索） |
| `/note [名前]` | `/n` | 連絡先にインタラクションメモを追加 |
| `/visit [名前]` | `/v` | 連絡先に訪問記録を追加 |
| `/a [名前]` | — | 連絡先に名刺写真を追加（OCR）；未登録の場合は新規作成可 |
| `/p [名前]` | — | 連絡先に集合写真を追加 |
| `/li` | `/linkedin` | LinkedIn プロフィールのスクリーンショットを送信、AI が連絡先として解析 |
| `/news` | — | ニュースレター素材を蓄積（`newsletter` 権限必要） |
| `/b [説明]` | `/batch` | バッチモード：連続で複数枚撮影、OCRはバックグラウンドで実行；出会った場所を説明として追記するとバッチ全体に適用 |
| `/done` | — | バッチモード終了、OCRをキューに追加 |
| `/cancel` | — | 現在のモードを中止 |
| `/email [キーワード]` | `/e` | 連絡先詳細からメールを送信 |
| `/work [説明]` | `/w` | AI が自然言語を解析してタスクを作成 |
| `/tasks` | `/t` | 自分の未処理タスクを一覧表示 |
| `/user` | `/u` | 組織のメンバー全員を一覧表示 |
| `/ai` | — | 現在使用中の AI モデルを表示 |
| `/stop` | — | （Super Admin）メンテナンスモードを有効化；`/stop off` で無効化 |

---

## コマンド詳細

### `/search` — 連絡先検索

```
/search 山田太郎
/s 山田
/s cancerfree
```

検索結果には名刺写真と以下のクイックボタンが表示されます：
- **📋 インタラクションログ** — 最新 5 件を表示、「さらに読み込む」に対応
- **✉️ メール送信** — メール送信フローへ移動
- **📝 メモ** — メモ記録フローへ移動

---

### `/note` / `/n` — インタラクションメモの追加

**方法 1：インタラクティブ**
```
/note または /n
→ Bot: 前回の連絡先に対してですか？（該当する場合）
→ 名前を選択または入力して検索
→ Bot: メモの内容を入力してください
→ 入力: 本日訪問、協力に前向きな姿勢
```

**方法 2：名前を指定してクイック検索**
```
/n 山田太郎
→ Bot: 見つかりました。メモ入力フローへ移動します
```

**連絡先が見つからない場合**：メモは未分類として自動保存されます。後で Web から分類できます。

**名前の省略**：最近操作した連絡先がある場合、名前を省略すると Bot が「○○さんに対してですか？」と確認します。

---

### `/visit` / `/v` — 訪問記録の追加

**方法 1：インタラクティブ**
```
/visit または /v
→ Bot: 前回の連絡先に訪問記録を追加しますか？（該当する場合）
→ 名前を選択または入力して検索
→ Bot: 訪問日時を入力してください（例：2026-03-29 14:00）、またはスキップと入力
→ Bot: 訪問場所を入力してください、またはスキップと入力
→ Bot: 訪問内容を入力してください
```

**方法 2：名前を指定してクイック検索**
```
/v 山田太郎
→ Bot: 見つかりました。日時入力フローへ移動します
```

---

### `/a` — 連絡先に名刺を追加

名刺写真を追加または置き換えたい場合に使用します。送信後、AI OCR が認識を行い、既存データとの差分を表示してユーザーが更新するかどうかを確認します。連絡先が存在しない場合は新規作成もできます。

**方法 1：前回の連絡先を使用**
```
/a
→ Bot: 前回の連絡先：山田太郎、名刺写真を送信してください  [⏭ スキップ、名刺不要]
→ 写真を送信
→ AI OCR → 差分を表示 → 確認/キャンセル
```

**方法 2：名前を指定**
```
/a 山田太郎
→ Bot: 見つかりました。名刺写真を送信してください  [⏭ スキップ、名刺不要]
```

**方法 3：名前 + 会社（未登録の場合は新規作成）**
```
/a 山田太郎 | ABC会社
→ Bot: 「山田太郎」が見つかりません。新規作成しますか？
   [✅ 「山田太郎 · ABC会社」を作成]  [❌ キャンセル]
→ 作成後：名刺写真を送信してください  [⏭ スキップ、名刺不要]
```

「⏭ スキップ、名刺不要」を押すと、名刺写真を省略して連絡先の作成を完了できます。

---

### `/news` — ニュースレター素材を蓄積

月内随時ニュースレターの素材を蓄積し、月末に管理者が「AI 編集」で 3 言語の draft campaigns を一括生成します。`newsletter` 権限が必要です。

```
/news
→ Bot: 「現在の期 2026-05。どのセクションに追加？」
   [📜 先月の振り返り]  [🔮 来月の予告]
→ セクション選択 → Bot: 「Story タイトル？」
→ 入力：AACR Taiwan Night
→ Bot: 「イベント日付？YYYY-MM-DD または「スキップ」」
→ 入力：2026-04-29
→ Bot: 「内容を貼り付け（テキスト＋写真）、/done で完了」
→ 写真を貼る、テキストを貼る（複数回 OK）
→ /done
→ Bot: ✅ 2026-05 先月の振り返りに保存しました
```

整理ページは `/admin/newsletter/draft/{period}` — 複数人で蓄積、編集、並べ替え、AI 編集が可能。

---

### `/p` — 連絡先に集合写真を追加

**方法 1：前回の連絡先を使用**
```
/p
→ Bot: 前回の連絡先：山田太郎、集合写真を送信してください
→ 写真を送信（推奨：長押し → ファイルとして送信すると撮影日時と GPS 情報が保持されます）
```

**方法 2：名前を指定**
```
/p 山田太郎
→ Bot: 見つかりました。集合写真を送信してください
```

---

### `/li` / `/linkedin` — LinkedIn スクリーンショットから連絡先を作成

```
/li または /linkedin
→ Bot: LinkedIn のプロフィールのスクリーンショットを送信してください
→ スクリーンショットを送信
→ AI が名前／役職／会社／Email／LinkedIn URL を解析
→ ✅ 追加を確認 / ❌ キャンセル
```

LinkedIn のスクリーンショットとして認識できない場合は、再送を促します。

---

### `/email` — メール送信

```
/email
→ 連絡先を検索 → 選択
→ 方法を選択：テンプレート / AI 生成
→ 内容を確認 → 送信
```

メールは Microsoft メールボックスから送信されます（Microsoft Graph API の認可が必要）。

---

### `/work` — タスク作成（AI 自然言語解析）

```
/work 来週金曜日までに山田さんに Q1 業績レポートをまとめてもらう
/w 明日の朝に田中さんに電話することを自分にリマインド
/w 鈴木さんと佐藤さんに今月末までに顧客リストを完成させてもらう
```

AI（Gemini）が解析するもの：
- **タイトル**：タスクの主な内容
- **期限**：「明日」「来週金曜日」「今月末」などの自然言語
- **担当者**：システムのメンバー一覧から名前で照合

---

### `/tasks` — 未処理タスクの確認

自分のすべての `pending` タスクを一覧表示します（自分が作成したもの + 自分に割り当てられたもの）。

各タスクにボタンが付きます：
- **✅ 完了** — 完了としてマーク
- **⏭ 延期** — 新しい期限を入力
- **❌ キャンセル** — タスクをキャンセル

---

### `/user` — 組織メンバー一覧

すべてのメンバーの名前・メールアドレス・Telegram ID を表示します（全メンバーが使用可能）。

---

### `/lang` — Bot の言語切り替え

```
/lang zh   → 繁體中文
/lang en   → English
/lang ja   → 日本語
```

Bot の返信言語のみに影響し、設定はアカウントに保存されます。

---

### `/ai` — 現在の AI モデルを確認

現在使用中の AI モデルの名称と提供元を表示します。未指定の場合はシステムのデフォルトモデル（gemini-2.5-flash）が使用されます。

---

### `/stop` — メンテナンスモード（Super Admin 限定）

`/stop` でメンテナンスモードを有効にすると、すべてのユーザーにメンテナンス中の案内が表示されます。`/stop off` で無効化して通常運用に戻ります。Super Admin のみ使用できます。

---

## 名刺の送信（コマンド不要）

Bot との会話に写真を直接送信します：

```
[写真を送信]
→ Bot: 認識中...
→ Bot: 認識結果：
       名前：山田太郎
       会社：ABC 株式会社
       ...
       保存しますか？
→ ✅ 保存 / ❌ 破棄
```

認識結果が誤っている場合は、保存後に Web で編集できます。

### 既存の連絡先を検出したとき（同一 email）

Bot は追加で 2 つのボタンを表示します：

| ボタン | 動作 | 使う場面 |
|---|---|---|
| 📌 「X」に追加 | **旧データを保持**：空欄には新しい値を入れ、競合する項目はインタラクションログに記録（上書きしない） | 同じ会社で再会し、既存の連絡先に新しい名刺を追加 |
| 🔄 「X」を更新（転職） | **新データで上書き**：競合する項目に新しい値を書き込み、**旧値は履歴としてインタラクションログに記録** | 相手が転職・転社し、最新情報に更新したい |
| ✅ それでも新規作成 | 強制的に新規作成（2 件の独立した連絡先） | 同姓同名の別人 |
| ❌ 破棄 | キャンセル | 撮り間違い／不要 |

## バッチ撮影（`/b` + `/done`）

一度に多くの名刺を処理したいとき（会議帰り、名刺の山）に便利です。1 枚ずつの同期 OCR とは流れが異なり、**Bot はその場では OCR せずにすぐ受け取る**ため、待たずに連続で撮影できます。バックグラウンドの OCR が完了すると通知され、Web の「審査待ち名刺」（`/contacts/pending`）でまとめて確認します。

```
/b
→ Bot: 📦 バッチモードに入りました
[撮影 #1] → Bot: 📥 1 枚目を受け取りました
[撮影 #2] → Bot: 📥 2 枚目を受け取りました
...
/done
→ Bot: ✅ 計 N 枚を OCR に送信、完了後に通知します
（バックグラウンドで認識中…）
→ Bot: ✅ N/M 枚の認識が完了、審査へ → /contacts/pending
```

**説明付きバッチ（`/b 説明`）**：`/b` のあとに、その人たちとどこで出会ったかを一文で追記すると、AI が「出会った場所／日付／紹介者」を解析し、このバッチの**すべて**の名刺に自動適用します。1 枚ずつ入力する手間が省けます。

```
/b 先週、東京の BioJapan で出会った
→ Bot: 🤖 「出会いの場」を解析中...
→ Bot: 📦 バッチモードに入りました（このバッチは自動タグ付け：📍 BioJapan／📅 2026-06-25）
[撮影 #1] → Bot: 📥 1 枚目を受け取りました
...
/done
```

途中で抜けたい場合：`/cancel`。受け取り済みの写真は**削除されません**。pending エリアに残り、Web で確認または削除できます。

認識に失敗した（名前が取れなかった）画像は自動的に「マイ失敗スキャン」（`/contacts/failed-scans`）へ移動します。そこで元画像の確認や削除ができます。再試行するには写真を撮り直してアップロードしてください。
