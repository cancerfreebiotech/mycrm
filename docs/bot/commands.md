## 指令詳細說明

### `/search` — 搜尋聯絡人

```
/search 王大華
/s 王
/s cancerfree
```

搜尋結果每筆顯示名片照片，附快速按鈕：
- **📋 互動紀錄** — 查看最新 5 筆，支援「載入更多」
- **✉️ 發信** — 跳轉到 Email 發送流程
- **📝 筆記** — 跳轉到記錄筆記流程

---

### `/note` / `/n` — 新增互動筆記

**方式一：互動式**
```
/note 或 /n
→ Bot: 要針對上一位聯絡人嗎？（若有）
→ 選擇或輸入姓名搜尋
→ Bot: 請輸入筆記內容
→ 輸入: 今天拜訪，有合作意願
```

**方式二：帶姓名快速搜尋**
```
/n 王大華
→ Bot: 找到後直接進入輸入筆記流程
```

**找不到聯絡人時**：筆記自動存為未歸類，之後可從 Web 歸類。

**省略姓名**：若最近有操作過某位聯絡人，可省略姓名，Bot 會提示「是否針對 XXX？」

---

### `/visit` / `/v` — 新增拜訪紀錄

**方式一：互動式**
```
/visit 或 /v
→ Bot: 要為上一位聯絡人新增拜訪紀錄嗎？（若有）
→ 選擇或輸入姓名搜尋
→ Bot: 請輸入拜訪日期時間（例：2026-03-29 14:00），或輸入「略過」
→ Bot: 請輸入拜訪地點，或輸入「略過」
→ Bot: 請輸入拜訪內容
```

**方式二：帶姓名快速搜尋**
```
/v 王大華
→ Bot: 找到後直接進入輸入日期時間流程
```

---

### `/a` — 為聯絡人新增名片

用於已有聯絡人但想補充/替換名片照片的情況。傳送後 AI OCR 辨識，顯示與現有資料的差異，由使用者確認是否更新。

**方式一：使用上一位聯絡人**
```
/a
→ Bot: 上一位聯絡人：王大華，請傳送名片照片  [⏭ 跳過，不需要名片]
→ 傳送照片
→ AI OCR → 顯示 diff → 確認/取消
```

**方式二：帶姓名指定**
```
/a 王大華
→ Bot: 找到後請傳送名片照片  [⏭ 跳過，不需要名片]
```

**方式三：帶姓名 + 公司（找不到時建立新聯絡人）**
```
/a 王大華 | ABC公司
→ Bot: 找不到「王大華」，要建立新聯絡人嗎？
   [✅ 建立「王大華 · ABC公司」]  [❌ 取消]
→ 建立後請傳送名片照片  [⏭ 跳過，不需要名片]
```

按「⏭ 跳過，不需要名片」可略過名片，直接完成聯絡人建立。

---

### `/news` — 累積電子報素材

整月隨手累積電子報素材，month-end 由 admin 用「AI 編寫」一鍵生成 3 個 draft campaigns。需要 `newsletter` 權限。

```
/news
→ Bot: 「當期 2026-05。要加到哪一段？」
   [📜 上月回顧]  [🔮 下月預告]
→ 選段落 → Bot: 「Story 標題？」
→ 輸入：AACR Taiwan Night
→ Bot: 「事件日期？YYYY-MM-DD 或「略過」」
→ 輸入：2026-04-29
→ Bot: 「請貼內容（文字+照片），完成輸入 /done」
→ 貼照片、貼文字（可多次）
→ /done
→ Bot: ✅ 已存到 2026-05 上月回顧
```

整理頁面在 `/admin/newsletter/draft/{period}`，多人協作累積、編輯、排序、AI 編寫。

---

### `/p` — 為聯絡人新增合照

**方式一：使用上一位聯絡人**
```
/p
→ Bot: 上一位聯絡人：王大華，請傳送合照
→ 傳送照片（建議長按 → 以檔案傳送，可保留拍攝時間和 GPS 地點）
```

**方式二：帶姓名指定**
```
/p 王大華
→ Bot: 找到後請傳送合照
```

### `/li` / `/linkedin` — 從 LinkedIn 截圖建立聯絡人

```
/li 或 /linkedin
→ Bot: 請傳送 LinkedIn 個人頁截圖
→ 傳送截圖
→ AI 解析姓名／職稱／公司／Email／LinkedIn 連結
→ ✅ 確認新增 / ❌ 取消
```

無法辨識為 LinkedIn 截圖時，會提示重新傳送。

### `/email` — 發送 Email

```
/email
→ 搜尋聯絡人 → 選擇
→ 選擇方式：範本 / AI 生成
→ 確認內容 → 發送
```

發信使用你的 Microsoft 信箱（需完成 Microsoft Graph API 授權）。

### `/work` — 建立任務（AI 自然語言解析）

```
/work 下週五前請陳小明整理 Q1 業績報告
/w 提醒自己明天早上打電話給王大華
/w 請李四和王五本月底前完成客戶名單
```

AI（Gemini）會解析：
- **標題**：任務主要內容
- **截止時間**：「明天」「下週五」「本月底」等自然語言
- **負責人**：從系統成員名單中比對姓名

---

### `/tasks` — 查看待處理任務

列出你的所有 `pending` 任務（我建立的 + 指派給我的）。

每筆任務附按鈕：
- **✅ 完成** — 標記完成
- **⏭ 延後** — 輸入新截止時間
- **❌ 取消** — 取消任務

---

### `/meet` / `/m` — 安排會議（AI 自然語言解析）

用自然語言安排會議。Bot 以 AI 解析時間、與會者、地點，回覆一張確認卡片，按「確認建立」才會真正寫入你的 Outlook／Teams 行事曆。

```
/meet 明天下午3點 和 Luna 開產品會議
/m 3/25 下午1點 參訪 九州大學實驗室
→ Bot: ⏳ AI 解析中...
→ Bot: [確認卡片]
        📅 產品會議
        🕐 3/25(二) 13:00 – 14:00（台北）
        👤 你、Luna
        [確認建立]  [取消]
→ 按「確認建立」→ Bot: ✅ 行程已建立！（附行事曆連結）
```

- 與會者從組織成員名單以姓名／Email 比對。
- 建立行程需要你已在 myCRM 網頁完成 Microsoft 登入授權。
- 按「取消」則不建立行程。

---

### `/met` — 批次標記「在哪認識」

為你最近建立的 N 位聯絡人，一次補上「在哪認識／日期／介紹人」。適合剛用 `/b` 批次掃完一疊名片後，統一標記整批的來源。

格式：`/met [數量] [描述]`（數量上限 20）

```
/met 5 上週在東京 BioJapan 認識，Luna 介紹
→ Bot: 🤖 解析中...
→ Bot: 找到最近 5 位聯絡人，將標記：
        📍 BioJapan  📅 2026-06-25  🤝 介紹人 Luna
        1. 王大華（ABC 公司）
        ...
        [確認套用]  [取消]
→ 按「確認套用」→ 整批寫入「在哪認識」資訊
```

AI 會從描述解析出「在哪認識／日期／介紹人」，套用到最近 N 位聯絡人。按「取消」則不變更。

## 傳送名片（無需指令）

直接在 Bot 對話中傳送圖片：

```
[傳送照片]
→ Bot: 辨識中...
→ Bot: 辨識結果：
       姓名：王大華
       公司：ABC 公司
       ...
       是否存檔？
→ ✅ 存檔 / ❌ 不存檔
```

若辨識結果有誤，存檔後可進入 Web 編輯。

### 偵測到既有聯絡人時（同 email）

Bot 會多顯示兩顆按鈕：

| 按鈕 | 行為 | 何時用 |
|---|---|---|
| 📌 加到「X」 | **保留舊資料**：空欄填入新值，衝突欄位寫入互動紀錄（不覆蓋） | 同公司、再次見面，加新名片到既有聯絡人 |
| 🔄 更新「X」（換工作） | **用新資料覆蓋**：衝突欄位寫入新值，**舊值寫入互動紀錄留歷史** | 對方換工作 / 換公司，要更新成最新資訊 |
| ✅ 仍建立新聯絡人 | 強制建新（兩個獨立聯絡人） | 同名不同人 |
| ❌ 不存檔 | 取消 | 拍錯 / 不需要 |

## 批次拍照（`/b` + `/done`）

適合一次有很多張要處理的時候（剛從會議回來、堆名片）。流程跟單張同步辨識不一樣：**Bot 立刻收下、不當下辨識**，使用者可以連續拍很多張不用等；後台辨識完成再通知，到 web 上的「待審核名片」（`/contacts/pending`）統一審核。

```
/b
→ Bot: 📦 已進入 batch 模式
[拍照 #1] → Bot: 📥 已收第 1 張
[拍照 #2] → Bot: 📥 已收第 2 張
...
/done
→ Bot: ✅ 共 N 張已送辨識，完成後通知你
（背景辨識中…）
→ Bot: ✅ 已完成辨識 N/M 張，前往審核 → /contacts/pending
```

**帶描述批次（`/b 描述`）**：在 `/b` 後面接一句話說明在哪認識這批人，AI 會解析出「在哪認識／日期／介紹人」，自動套用到這批**每一張**名片，省得逐張補填。

```
/b 上週在東京 BioJapan 認識
→ Bot: 🤖 解析「在哪裡遇見」...
→ Bot: 📦 已進入 batch 模式（這批會自動標記：📍 BioJapan／📅 2026-06-25）
[拍照 #1] → Bot: 📥 已收第 1 張
...
/done
```

中途想退出：`/cancel`，已收的照片**不會刪除**，留在 pending 區可在 web 確認或刪除。

辨識失敗（沒抓到姓名）的圖會自動移到「我的失敗辨識」（`/contacts/failed-scans`），可在那裡看原圖或刪除；想重試請重新拍照上傳。
