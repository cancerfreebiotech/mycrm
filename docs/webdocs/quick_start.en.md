## What Is myCRM

myCRM is the company-wide business-card and customer-relationship system for cancerfree.io. The core flow is simple: snap a business card in Telegram → AI extracts the name, company, title, email, and other fields → it lands in the CRM. From the web interface you can then search and manage contacts, log visits and notes, assign tasks, and send email marketing campaigns. This page walks you through everything from your first login to getting up and running.

## Sign In with Your Microsoft Account

myCRM uses **Microsoft Account (Azure AD) Single Sign-On (SSO)**, restricted to company accounts on the `@cancerfree.io` domain. There is no separate password to set.

1. Open `crm.cancerfree.io` in your browser
2. Click **"Sign in with Microsoft Account"**
3. Enter your company email and password, and approve the application authorization
4. Your first login automatically creates your account (role defaults to `member`) and then prompts you to set up MFA

> If you see "Sign-in restricted to @cancerfree.io accounts", you used a non-company account. Sign in again with your company email.

## Set Up and Use MFA (Two-Factor Authentication)

For account security, **every user must complete MFA (two-factor authentication)**. No feature is usable until it is done.

### First-Time MFA Setup

After your first login the system automatically redirects you to the MFA setup page:

1. Install an authenticator app on your phone (Google Authenticator, Microsoft Authenticator, 1Password, etc. — any will do)
2. **Scan the on-screen QR code** with the app; if you can't scan it, enter the **manual secret key** shown below the code instead
3. Enter the **6-digit code** generated by the app to finish enrollment
4. Once enrolled, you can use the system normally

### Every Subsequent Login

After enrollment, every login asks for the current **6-digit code** from your authenticator before letting you in. The code refreshes every 30 seconds, so enter the latest one.

### Changing Phones or Losing Your Authenticator

- To switch devices or re-enroll: top-right corner → **Settings** → the MFA section.
- If you lose your phone, change devices, or can't get a code: contact a Super Admin to reset your MFA on the **User Management** page. After the reset, sign in again to enroll your new device.

## Interface Tour

After login, the sidebar sits on the left. The top section holds core features every member can use; items below the divider are advanced features that require a grant, plus Super Admin-only items. A floating **AI Assistant** button is at the bottom-right of every page and can be opened anywhere.

### Core Features (Available to All Members)

| Item | Description |
|---|---|
| Dashboard | Dashboard and stats |
| Contacts | Business-card contact list |
| Pending Cards | Cards awaiting your review after AI recognition |
| My Failed Scans | Cards you scanned that failed recognition |
| Photos | Card and event photos |
| Search Notes | Search contact notes |
| Tasks | Task list |
| Reports | Statistics reports |
| Email Campaigns | Email marketing campaigns |
| Settings | Profile, language, theme, MFA, Telegram linking |
| Documentation | This documentation |
| 💬 Feedback | Send feedback |

### Advanced Features (Granted Per-Feature by a Super Admin)

These items are **always visible** in the sidebar, but each one must be granted individually by a Super Admin on the **User Management** page before you can use it — opening a page without the grant shows an insufficient-permission notice: Tag Management, Unassigned Notes, Email Templates, Prompt Management, Country Management, Newsletter, Failed Scans, Duplicates, Shared Emails, Email Recovery, CamCard Import, Trash, and User Management.

### Super Admin Only

Visible only to Super Admins: Org Settings, AI Model Management, System Health, MCP Tokens, MCP Activity, Audit Log, Suppressions, DSAR Lookup, Newsletter Overview, and Feedback Management.

### Using the Sidebar

- **Mobile:** toggle the sidebar with the hamburger menu at the top-left.
- **Desktop:** it expands on hover, and the arrow at the bottom-left pins it collapsed.
- All touch targets are ≥ 44px for one-handed mobile use.

## Language and Theme

### Switching Language (Traditional Chinese / English / Japanese)

Either method works; your choice is saved to your account and applied automatically on next login:

- Click the 🌐 icon in the top-right of the header → select a language
- **Settings** → Language selection

### Dark / Light Theme

- The ☀️ / 🌙 icon in the top-right of the header toggles theme instantly
- You can also set it in **Settings** → Theme selection
- The default is Light; you can switch to Dark, and your preference is remembered

## Linking the Telegram Bot

To scan cards, take notes, and manage tasks from Telegram, link your Telegram account first. It works like this: put your **numeric Telegram ID** into Settings, and the Bot recognizes you by that ID whenever you message it.

### Step 1: Get Your Telegram ID

Search for **`@userinfobot`** on Telegram and send any message. It replies with your numeric ID (e.g. `123456789`).

### Step 2: Enter It in Settings

Log in to myCRM → top-right corner → **Settings** → paste your Telegram ID and save. The ID must be numeric; if you see "already linked to another member", that ID is already in use — double-check you entered the right one.

### Step 3: Start Using the Bot

Search for **`@CF_CRMBot`** (CancerFreeCRM) on Telegram and start a chat. Send **`/help`** to see every available command, or just send a card photo to test recognition. If the Bot replies "You are not authorized", the ID isn't linked correctly yet — go back to Settings and confirm it.

> To receive task notifications, you can also link the Microsoft Teams Bot: open a 1-on-1 chat with the Bot in Teams and send any message (e.g. `help`). The Bot auto-links using your company account, and you'll then get notification cards whenever a task is assigned to you.

## Having Trouble? Send Feedback

The **💬 Feedback** item in the sidebar (`/feedback`) lets you report issues or request features anytime:

1. Choose a type: **Feature Request** or **Bug Report**
2. Fill in a title and description (both required)
3. Optionally attach a screenshot (recommended for bug reports)
4. After you submit, an admin tracks and handles it on the backend **Feedback Management** page

## Roles and Permissions

myCRM has only two roles:

| Role | Description |
|---|---|
| `member` | General user. Can use all core features; advanced features require a separate grant. |
| `super_admin` | System administrator. Has every feature, and can manage users, assign grants, reset other users' MFA, and manage AI models and system settings. |

Advanced features use **per-feature grants**: a Super Admin enables the needed features (e.g. Newsletter, Tag Management) for an individual member on the **User Management** page, and that member can then use them without any role change.
