# 使用者管理

路徑：`/admin/users`

存取權限：`super_admin`，或已被授權「使用者管理」功能的一般成員。被授權的成員只能**重設 MFA** 與**編輯 Telegram ID**，無法變更角色、調整功能授權或切換維護模式。

---

## 使用者列表

顯示所有已登入系統的成員。欄位皆可點擊表頭排序：

| 欄位 | 說明 |
|------|------|
| 姓名 | 使用者設定的顯示名稱 |
| Email | Microsoft 帳號（主鍵） |
| Telegram | 已綁定的 Telegram ID，可直接編輯 |
| Teams | 是否已綁定 Microsoft Teams |
| 角色 | member / super_admin |
| 最後登入 | 最近一次登入時間 |
| MFA | 是否已設定多因素驗證 |

手機版以卡片方式呈現同樣的資訊。

---

## 角色管理

> 僅 super_admin 可操作。

點擊該成員的「升為 Super Admin」／「降為一般成員」按鈕切換角色。

- `member` → 一般使用者
- `super_admin` → 可存取全部管理功能

> 無法修改自己的角色；且至少需保留一位 super_admin，避免所有管理員被降權。

---

## 帳號停用 / 離職

> 僅 super_admin 可操作。

每位成員的角色標籤旁會顯示帳號狀態徽章：**啟用**（綠色）或**已停用**（紅色）。

點該成員的「停用」按鈕即可停用帳號（需二次確認）；已停用者按「重新啟用」可恢復。

- 停用**在該成員下次登入時生效**：他會在登入時被登出並擋在系統外，無法使用任何功能。
- 目前已登入的 session 不會立即中斷，會維持到自然到期（最長 7 天）。
- **Super Admin（`pohan.chen@cancerfree.io`）不可被停用**。

---

## 功能授權

> 僅 super_admin 可操作，且只對 member 顯示。

一般成員的列會顯示一排功能權限標籤（chips），點擊即可授權或取消。已授權的項目會顯示 ✓。可授權的功能包含：標籤、郵件範本、提示詞、國家、電子報、辨識失敗審查、重複聯絡人、名片王匯入、回收區、使用者管理，以及未分配筆記、匯出聯絡人、群發郵件等。

被授權的成員會在側邊欄看到對應項目。

---

## MFA 狀態與重設

MFA 欄顯示成員是否已完成多因素驗證（已設定／未設定）。

若成員已設定過任何驗證器，該列會出現「重設」按鈕。點擊後需二次確認；重設會刪除該成員現有的所有 MFA 驗證器，成員**下次登入時必須重新設定** MFA。

此操作 super_admin 與被授權「使用者管理」的成員皆可執行。

---

## 編輯 Telegram ID

在 Telegram 欄點擊 ✏️ 進入行內編輯，輸入數字後儲存：

- 必須為正整數，否則會擋下。
- 若少於 9 位數（一般 Telegram ID 為 9–10 位）會跳出提醒，確認後才會儲存。
- 清空後儲存則解除綁定。

此操作 super_admin 與被授權「使用者管理」的成員皆可執行。

---

## Teams 綁定

Teams 欄僅顯示綁定狀態（已綁定／未設定），為唯讀。綁定由成員自行透過 Teams 連結帳號後產生。

---

## 維護模式

> 僅 super_admin 可操作。

頁面頂端提供維護模式開關。啟用後，一般使用者會被導向維護頁，僅 super_admin 可繼續使用系統；啟用前需二次確認。

---

## 注意事項

- 使用者帳號在**第一次登入**時自動建立，無法手動預先建立
- 只有 `@cancerfree.io` 網域的 Microsoft 帳號才能登入
- 刪除使用者：目前需透過 Supabase Dashboard 直接操作 DB
