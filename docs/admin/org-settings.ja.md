# 組織設定とブランド

場所：`/admin/org-settings`（super_admin のみ）

コード変更や再デプロイなしで、全社共通の組織情報とニュースレターのブランドを管理します。

## 設定項目

| 項目 | 用途 |
|------|------|
| 組織名 | システムに表示される会社名 |
| ログイン許可メールドメイン | バックエンドのログインドメイン検証元（カンマ区切り） |
| ニュースレターのロゴ URL | ニュースレター骨組みのロゴ |
| ニュースレターの返信先メール | ニュースレターの reply-to |
| 会社ウェブサイト / Facebook / LinkedIn | ニュースレターフッターのソーシャルリンク |
| フィードバック通知の宛先 | 毎日のシステムフィードバック要約メールの宛先 |
| 送信者の表示名 | メール送信者として表示される名前（SendGrid from name） |
| 社内メールドメイン | 会議出席者が社内かどうかの判定に使うドメイン（@ なし） |
| 組織メールドメイン | 受信メールの帰属判定に使う会社ドメイン（@ なし） |
| BCC アーカイブ受信ドメイン | メール自動アーカイブの BCC 受信ドメイン（@ なし） |
| 郵送先住所 | ニュースレターフッターに表示、CAN-SPAM 準拠 |
| 組織オーナーのメール | システム警告・帰属不明の受信メール・システムメールの既定返信先の宛先 |
| アプリケーション URL | メールや要約内のリンクに使う対外 URL（例：`https://crm.cancerfree.io`） |

> 表の「送信者の表示名」から「アプリケーション URL」までの 7 項目（メール／システムパラメータ）は **v7.9.4** からここで設定できます。以前はコードに直書きされていました。

## モジュール切り替え

機能モジュール全体をワンタッチで無効化できます。変更は同じく最大 60 秒で反映されます：

| 切り替え | オフ時の影響 |
|------|------|
| Hunter.io 自動補完 | 新しい連絡先が Hunter.io で会社・役職を自動補完されなくなり、[システム健全性](health.md)ページで補完を実行すると「モジュール無効」の通知が表示されます |
| AI アシスタント | AI アシスタントのチャット（ウェブ版と Telegram `/ai`）が停止します |

## 使い方

1. 項目を編集するか切り替えを操作して「保存」をクリック。
2. 設定は **60 秒キャッシュ**され、保存後最大 60 秒で反映されます。
3. 空欄はシステムのデフォルト値を使用します（項目のヒントに表示）。

> すべての変更は監査ログに記録されます（アクション：組織設定変更）。
