# 管理者

> このエリアはシステム管理機能で、ほとんどは `super_admin` に限定されます。v2.3 以降、一部の機能は super_admin が[ユーザー管理](users.md)で一般メンバーに個別に権限付与できます。権限を付与されていないメンバーには、該当項目がサイドバーに表示されません。

**一般メンバーに付与可能な機能**：タグ、メールテンプレート、プロンプト、国、ニュースレター、スキャン失敗審査、重複連絡先、名刺インポート、ゴミ箱、ユーザー管理（MFA リセット / Telegram 編集のみ）。

**レポート**はすべてのメンバーが利用でき、各自が自分のスケジュールを管理します。super_admin はすべてのスケジュールを確認できます。

**super_admin のみ**：AI モデル、システム健全性、MCP アクティビティ、MCP トークン、メール復活、共用 Email、フィードバック。

| 機能 | 説明 |
|------|------|
| ユーザー管理 | メンバーアカウント・ロール・機能権限・MFA リセット・Telegram ID・Teams 連携状態・メンテナンスモードを管理 |
| タグ管理 | 連絡先タグの追加・名称変更・削除、および Email ブラックリスト指定 |
| AI モデル | AI プロバイダー（エンドポイント）と利用可能なモデルの設定 |
| メールテンプレート | AI によるメール生成のプロンプトテンプレートを管理 |
| プロンプト | 名刺認識と AI アシスタント用のシステムプロンプトを設定（空白でシステムデフォルト） |
| 国管理 | 国リストの管理（ISO コード・多言語名・国旗 emoji・有効状態） |
| ニュースレター | 購読者管理 / キャンペーン編集 / 送信 / PDF 出力 / RSS feed（Substack 自動下書き用） |
| レポート | 交流記録レポートの生成（JSON プレビューまたは Excel）と定期スケジュール送信 |
| 名刺インポート審査 | 名刺を一括インポートし、会社ごとに認識結果を審査 |
| 重複連絡先審査 | 重複した連絡先を検出して統合 |
| スキャン失敗審査 | 認識に失敗した（氏名を識別できない）名刺を確認し、手動で連絡先を作成後に完了マーク |
| 未割り当てメモ | 連絡先に紐付いていない Bot メモの確認 |
| ゴミ箱 | 削除された連絡先を復元または完全削除 |
| メール復活 | バウンス・無効メールの連絡先について、転職後の新しい名刺を見つけワンクリックでメールを置換 |
| 共用 Email の連絡先 | 同じ email を 2 件以上の連絡先で共有しているケースを検出（夫婦・家族・共用受信箱） |
| フィードバック管理 | ユーザーから送信されたフィードバックを確認・処理 |
| システム健全性 | 外部サービスとシステムコンポーネントの健全性を確認 |
| MCP アクティビティログ | AI アシスタント（MCP）の操作ログを確認 |
| MCP トークン | 外部エージェント用のアクセスキーを管理（平文は一度だけ表示） |
