# 管理者

> このエリアはシステム管理機能で、ほとんどは `super_admin` に限定されます。v2.3 以降、一部の機能は super_admin が[ユーザー管理](users.md)で一般メンバーに個別に権限付与できます。権限を付与されていないメンバーには、該当項目がサイドバーに表示されません。

**一般メンバーに付与可能な機能**：タグ、未割り当てメモ、メールテンプレート、プロンプト、国、ニュースレター、スキャン失敗審査、重複連絡先、名刺インポート、ゴミ箱、連絡先エクスポート、一括メール、ユーザー管理（MFA リセット / Telegram 編集のみ）。

**レポート**はすべてのメンバーが利用でき、各自が自分のスケジュールを管理します。super_admin はすべてのスケジュールを確認できます。

**super_admin のみ**：AI モデル、システム健全性、MCP アクティビティ、MCP トークン、メール復活、共用 Email、フィードバック、配信停止管理、DSAR照会、ニュースレター概要。

| 機能 | 説明 |
|------|------|
| [ユーザー管理](users.md) | メンバーアカウント・ロール・機能権限・MFA リセット・Telegram ID・Teams 連携状態・メンテナンスモードを管理 |
| [タグ管理](tags.md) | 連絡先タグの追加・名称変更・削除、および Email ブラックリスト指定 |
| [AI モデル](models.md) | AI プロバイダー（エンドポイント）と利用可能なモデルの設定 |
| [メールテンプレート](templates.md) | AI によるメール生成のプロンプトテンプレートを管理 |
| [プロンプト](prompts.md) | 名刺認識と AI アシスタント用のシステムプロンプトを設定（空白でシステムデフォルト） |
| [国管理](countries.md) | 国リストの管理（ISO コード・多言語名・国旗 emoji・有効状態） |
| ニュースレター | 購読者管理 / キャンペーン編集 / 送信 / PDF 出力 / RSS feed（Substack 自動下書き用） |
| [レポート](reports.md) | 交流記録レポートの生成（JSON プレビューまたは Excel）と定期スケジュール送信 |
| [名刺インポート審査](camcard.md) | 名刺を一括インポートし、会社ごとに認識結果を審査 |
| [重複連絡先審査](duplicates.md) | 重複した連絡先を検出して統合 |
| [スキャン失敗審査](failed-scans.md) | 認識に失敗した（氏名を識別できない）名刺を確認し、手動で連絡先を作成後に完了マーク |
| 未割り当てメモ | 連絡先に紐付いていない Bot メモの確認 |
| [ゴミ箱](trash.md) | 削除された連絡先を復元または完全削除 |
| [メール復活](email-recovery.md) | バウンス・無効メールの連絡先について、転職後の新しい名刺を見つけワンクリックでメールを置換 |
| [共用 Email の連絡先](shared-emails.md) | 同じ email を 2 件以上の連絡先で共有しているケースを検出（夫婦・家族・共用受信箱） |
| [フィードバック管理](feedback.md) | ユーザーから送信されたフィードバックを確認・処理 |
| [システム健全性](health.md) | 外部サービスとシステムコンポーネントの健全性を確認 |
| [MCP アクティビティログ](mcp-activity.md) | AI アシスタント（MCP）の操作ログを確認 |
| [MCP トークン](mcp-tokens.md) | 外部エージェント用のアクセスキーを管理（平文は一度だけ表示） |
| [監査ログ](audit-log.md) | ロール変更・削除などの特権操作の監査ログを確認。操作者／アクション／日付で絞り込み、CSV エクスポートも可能 |
| [組織設定とブランド](org-settings.md) | 全社の名称・ログインドメイン・ニュースレターブランド・フィードバック通知の宛先、およびモジュール切り替え（Hunter 補完・AI アシスタントをワンクリックで無効化） |
| 配信停止管理 | 任意の email を入力すると「送信できるか・なぜできないか」を即座に確認でき、5 つの抑制ソース（配信停止・ブラックリスト・バウンスなど）を集約 |
| DSAR照会 | email をキーに関連するすべての連絡先（削除済みを含む）と個人データの足跡を集約。ワンクリックでエクスポートでき、照会は自動的に監査記録に残ります |
| ニュースレター概要 | 直近 12 回の配信の開封／クリックのトレンド、リストごとの健全性（180 日間未開封を含む）、購読／配信停止／ブラックリストの総数 |
