# 組織設定與品牌

位置：`/admin/org-settings`（僅 super_admin）

管理全公司共用的組織資訊與電子報品牌，不需改程式或重新部署。

## 可設定項目

| 項目 | 用途 |
|------|------|
| 組織名稱 | 系統顯示的公司名稱 |
| 允許登入的 Email 網域 | 後端登入網域驗證來源（逗號分隔多網域） |
| 電子報 Logo 網址 | 電子報骨架的 Logo |
| 電子報回信 Email | 電子報的 reply-to |
| 公司網站 / Facebook / LinkedIn | 電子報頁尾的社群連結 |
| 系統回報通知收件人 | 每日系統回報摘要 Email 的收件人 |
| 寄件人顯示名稱 | 電子郵件寄件者顯示的名稱（SendGrid from name） |
| 內部 Email 網域 | 判斷與會者是否為內部同仁所用的網域（不含 @） |
| 組織 Email 網域 | 收信歸屬判斷所用的公司網域（不含 @） |
| BCC 歸檔收件網域 | 郵件自動歸檔的 BCC 收件匣網域（不含 @） |
| 實體郵寄地址 | 顯示於電子報頁尾，符合 CAN-SPAM 規定 |
| 組織擁有者 Email | 系統告警、無主歸屬信件與系統信預設回信的收件人 |
| 應用程式網址 | 信件與摘要中連結所用的對外網址（例如 `https://crm.cancerfree.io`） |

> 表中「寄件人顯示名稱」到「應用程式網址」等 7 項郵件／系統參數自 **v7.9.4** 起可在此設定，先前寫死於程式。

## 模組開關

一鍵停用整個功能模組，變更同樣最多 60 秒內生效：

| 開關 | 關閉後的影響 |
|------|------|
| Hunter.io 自動補全 | 新聯絡人不會自動透過 Hunter.io 補全公司與職稱；[系統健康](health.md)頁執行補查時會顯示「模組已停用」提示 |
| AI 助理 | AI 助理聊天功能（網頁版與 Telegram `/ai`）暫停使用 |

## 使用方式

1. 修改欄位或切換開關後按「儲存」。
2. 設定有 **60 秒快取**，儲存後最多 60 秒內生效。
3. 欄位留空代表使用系統預設值（顯示於欄位提示）。

> 所有變更都會寫入稽核日誌（動作：組織設定變更）。
