# AI 模型管理

路徑：`/admin/models`（僅 super_admin 可見）

---

## 架構說明

AI 模型採用兩層結構：

```
AI 服務商（Endpoint）
  └── AI 模型（Model）
        ├── gemini-2.0-flash
        └── gemini-1.5-pro
```

使用者在個人設定中，先選服務商、再選模型。

---

## 管理 AI 服務商

### 新增服務商

| 欄位 | 說明 | 範例 |
|------|------|------|
| 名稱 | 顯示名稱 | Google Gemini |
| Base URL | API 端點 | `https://generativelanguage.googleapis.com/v1beta` |
| API Key | 服務商的 API Key | `AIza...` |

### 編輯 / 刪除

點擊服務商列表的 ✏️ / 🗑 按鈕操作。

> 刪除服務商前請先刪除底下的所有模型。

---

## 管理 AI 模型

### 新增模型

在服務商底下點擊「+ 新增模型」：

| 欄位 | 說明 | 範例 |
|------|------|------|
| 模型 ID | API 使用的 model 名稱 | `gemini-2.0-flash` |
| 顯示名稱 | 使用者選擇時看到的名稱 | `Gemini 2.0 Flash` |
| 狀態 | 啟用 / 停用 | — |

### 停用模型

停用後，個人設定中的 dropdown 不再顯示此模型。已選擇此模型的使用者，系統會使用預設模型。

---

## 使用情境

| 操作 | 使用的模型 |
|------|------------|
| Bot 名片 OCR | 使用者個人設定的模型 |
| Web 上傳名片辨識 | 使用者個人設定的模型 |
| 任務自然語言解析 `/work` | 使用者個人設定的模型 |
| AI 郵件生成 | 使用者個人設定的模型 |
