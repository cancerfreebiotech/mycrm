# AI 模型管理

路徑：`/admin/models`（僅 super_admin 可見）

---

## 架構說明

AI 模型採用兩層結構：

```
AI 服務商（Endpoint）
  └── AI 模型（Model）
        ├── gemini-2.0-flash
        └── gemini-1.5-pro
```

使用者在個人設定中，先選服務商、再選模型。

---

## 管理 AI 服務商

服務商以列表呈現，欄位包含：服務商名稱、Base URL、API Key、Models（該服務商底下的模型數）、狀態。點擊任一列即可在下方展開該服務商的模型清單。

### 新增服務商

點「新增 Endpoint」，填寫：

| 欄位 | 說明 | 範例 |
|------|------|------|
| 服務商名稱 | 顯示名稱 | Google Gemini |
| Base URL | API 端點 | `https://generativelanguage.googleapis.com/v1beta` |
| API Key | 服務商的 API Key | `AIza...` |

### 更換 API Key

API Key 在列表中以遮蔽方式顯示，可點眼睛圖示切換顯示明文。點「更換 API Key」會出現行內輸入框，輸入新金鑰後儲存即可。（服務商名稱與 Base URL 不提供修改）

### 啟用 / 停用服務商

點狀態欄的切換鈕即可啟用或停用整個服務商。

### 刪除服務商

點該列的 🗑 並確認後，即可**直接刪除**服務商，不需先刪除底下的模型。

---

## 管理 AI 模型

先在上方點選一個服務商，下方才會出現其模型清單。

### 新增模型

在該服務商的模型區點「新增 Model」：

| 欄位 | 說明 | 範例 |
|------|------|------|
| Model ID | API 使用的 model 名稱 | `gemini-2.5-flash` |
| 顯示名稱 | 使用者選擇時看到的名稱 | `Gemini 2.5 Flash` |

### 啟用 / 停用模型

點模型列狀態欄的切換鈕切換。停用後，個人設定中的下拉選單不再顯示此模型；已選擇此模型的使用者，系統會改用預設模型。

### 刪除模型

點模型列的 🗑 並確認後刪除。

## 使用情境

| 操作 | 使用的模型 |
|------|------------|
| Bot 名片 OCR | 使用者個人設定的模型 |
| Web 上傳名片辨識 | 使用者個人設定的模型 |
| 任務自然語言解析 `/work` | 使用者個人設定的模型 |
| AI 郵件生成 | 使用者個人設定的模型 |
