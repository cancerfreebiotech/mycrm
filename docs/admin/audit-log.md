# 稽核日誌

路徑：`/admin/audit-log`（僅 `super_admin`；v7.4.0 新增）

---

## 這頁在做什麼

檢視系統中「敏感／特權操作」的稽核紀錄（例如角色變更、刪除等），供事後追查。資料來自 `admin_actions` 表，每頁 20 筆，可上下頁翻閱、依操作者／動作／日期篩選並匯出 CSV；右上「重新整理」可重新載入。

---

## 表格內容

| 欄位 | 說明 |
|------|------|
| 時間 | 操作發生時間（台北時區） |
| 操作者 | 執行者的 email |
| 動作 | 操作類型（會翻譯成可讀標籤） |
| 對象 | 被操作的目標（如聯絡人或使用者 ID） |
| 詳情 | 額外資料，點「展開」以 JSON 檢視 |

---

## 篩選與匯出

頂端的篩選列可縮小範圍，任一條件改變即時套用：

| 篩選 | 說明 |
|------|------|
| 操作者 | 以 email 關鍵字搜尋執行者 |
| 動作 | 下拉選單，涵蓋全部 19 種特權動作（見下） |
| 起始 / 結束日期 | 依**台北時區**的日界篩選 |

- 有套用任何篩選時會出現「清除篩選」。
- 「匯出 CSV」依目前篩選匯出（最多 5000 筆，含 UTF-8 BOM，Excel 可正確開啟中日文）。

### 可篩選的動作

自 v7.9.5 起，19 種特權動作全數有可讀標籤，與後端寫入點一一對應：重設 MFA、設定 Telegram ID、維護模式切換、永久刪除聯絡人、批次永久刪除、建立 MCP Token、撤銷 MCP Token、套用 Email 復原、Hunter 設定變更、設定 Webhook、發送更新通知、合併聯絡人、DSAR 資料查詢、設定帳號狀態、發送電子報、GDPR 資料匯出、設定角色、設定功能權限、組織設定變更。

---

## 注意事項

- 本頁僅 super_admin 可用，後端 API 也會再次驗證權限。
- 這裡是唯讀紀錄，不能在此修改或刪除紀錄。
