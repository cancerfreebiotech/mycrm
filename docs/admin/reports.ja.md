# レポート管理

パス：`/admin/reports`

アクセス権限：**すべてのメンバー**がレポートページを開き、レポートをその場で生成し、自分が作成したスケジュールを管理できます。`super_admin` はさらに「作成者」カラムが表示され、全員のスケジュールを確認できます。

機能の詳細は [機能説明 → レポート](../features/reports.md) をご覧ください。

---

## レポートをその場で生成

1. レポートの期間（開始・終了）を選択します。
2. 必要に応じてフィルターを適用：タグ、交流タイプ（会議／メモ／email）、作成者、国。
3. 「レポート生成」をクリック → ページ下部に交流記録がソート可能な表でプレビューされます。
4. または「Excel ダウンロード」をクリック → `.xlsx` ファイルを直接ダウンロードします。

---

## 定期レポートスケジュール

「スケジュールを追加」をクリックして設定フォームを開きます：

| フィールド | 説明 |
|-----------|------|
| スケジュール名 | スケジュールを識別する名前 |
| 頻度 | 毎週 / 毎月 / カスタム（Cron 式） |
| レポート期間（日数） | 各送信でカバーする直近の日数 |
| 受信者 | カンマ区切りのメールアドレス一覧 |

各スケジュールは有効／無効の切り替え、編集、削除ができます。「前回実行」カラムには、直近の自動送信の日時と結果（✅ 成功／⚠️ 失敗）が表示されます。

---

## スケジュール送信の仕組み

スケジュール送信は**バックエンドの自動処理**が担当します：システムは毎時チェックを行い、実行時期が来たスケジュールを自動的にそのスケジュールの受信者一覧へメール送信し、「前回実行」の状態を更新します。

手動トリガーは不要で、**外部メールアカウントを連携する必要もありません**。
