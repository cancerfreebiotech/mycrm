# AI モデル管理

パス：`/admin/models`（super_admin のみ表示）

---

## アーキテクチャの説明

AI モデルは 2 層構造を採用しています：

```
AI プロバイダー（Endpoint）
  └── AI モデル
        ├── gemini-2.0-flash
        └── gemini-1.5-pro
```

ユーザーは個人設定でまずプロバイダーを選択し、次にそのプロバイダー配下のモデルを選択します。

---

## AI プロバイダーの管理

プロバイダーは一覧で表示され、カラムはプロバイダー名・Base URL・API Key・Models（そのプロバイダー配下のモデル数）・状態です。いずれかの行をクリックすると、下部にそのプロバイダーのモデル一覧が展開されます。

### プロバイダーの追加

「エンドポイントを追加」をクリックし、次を入力します：

| フィールド | 説明 | 例 |
|-----------|------|-----|
| プロバイダー名 | 表示名 | Google Gemini |
| Base URL | API エンドポイント | `https://generativelanguage.googleapis.com/v1beta` |
| API Key | プロバイダーの API Key | `AIza...` |

### API キーの変更

API Key は一覧でマスク表示されます。目のアイコンをクリックすると平文の表示を切り替えられます。「API キーを変更」をクリックするとインライン入力欄が表示されるので、新しいキーを入力して保存します。（プロバイダー名と Base URL は編集できません）

### プロバイダーの有効化 / 無効化

状態カラムの切り替えボタンをクリックすると、プロバイダー全体を有効・無効にできます。

### プロバイダーの削除

その行の 🗑 をクリックして確認すると、プロバイダーを**直接削除**できます。配下のモデルを先に削除する必要はありません。

---

## AI モデルの管理

先に上部でプロバイダーを選択すると、下部にそのモデル一覧が表示されます。

### モデルの追加

そのプロバイダーのモデル欄で「モデルを追加」をクリックします：

| フィールド | 説明 | 例 |
|-----------|------|-----|
| モデル ID | API で使用するモデル名 | `gemini-2.5-flash` |
| 表示名 | ユーザーが選択時に表示される名前 | `Gemini 2.5 Flash` |

### モデルの有効化 / 無効化

モデル行の状態カラムの切り替えボタンをクリックします。無効化すると、個人設定のドロップダウンにこのモデルが表示されなくなり、すでに選択しているユーザーはデフォルトモデルにフォールバックします。

### モデルの削除

モデル行の 🗑 をクリックして確認すると削除されます。

## 使用シナリオ

| 操作 | 使用モデル |
|------|-----------|
| Bot 名刺 OCR | ユーザーの個人設定モデル |
| Web 名刺アップロード認識 | ユーザーの個人設定モデル |
| タスク自然言語解析 `/work` | ユーザーの個人設定モデル |
| AI メール生成 | ユーザーの個人設定モデル |
