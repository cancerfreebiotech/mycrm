# AI モデル管理

パス：`/admin/models`（super_admin のみ表示）

---

## アーキテクチャの説明

AI モデルは 2 層構造を採用しています：

```
AI プロバイダー（Endpoint）
  └── AI モデル
        ├── gemini-2.0-flash
        └── gemini-1.5-pro
```

ユーザーは個人設定でまずプロバイダーを選択し、次にそのプロバイダー配下のモデルを選択します。

---

## AI プロバイダーの管理

### プロバイダーの追加

| フィールド | 説明 | 例 |
|-----------|------|-----|
| 名前 | 表示名 | Google Gemini |
| Base URL | API エンドポイント | `https://generativelanguage.googleapis.com/v1beta` |
| API Key | プロバイダーの API Key | `AIza...` |

### 編集 / 削除

プロバイダー一覧の ✏️ / 🗑 ボタンをクリックして操作します。

> プロバイダーを削除する前に、配下のすべてのモデルを先に削除してください。

---

## AI モデルの管理

### モデルの追加

プロバイダー配下の「+ モデルを追加」をクリックします：

| フィールド | 説明 | 例 |
|-----------|------|-----|
| モデル ID | API で使用するモデル名 | `gemini-2.0-flash` |
| 表示名 | ユーザーが選択時に表示される名前 | `Gemini 2.0 Flash` |
| ステータス | 有効 / 無効 | — |

### モデルの無効化

無効化すると、個人設定のドロップダウンにこのモデルが表示されなくなります。すでにこのモデルを選択しているユーザーは、システムがデフォルトモデルを使用します。

---

## 使用シナリオ

| 操作 | 使用モデル |
|------|-----------|
| Bot 名刺 OCR | ユーザーの個人設定モデル |
| Web 名刺アップロード認識 | ユーザーの個人設定モデル |
| タスク自然言語解析 `/work` | ユーザーの個人設定モデル |
| AI メール生成 | ユーザーの個人設定モデル |
