# データ主体照会 (DSAR)

パス：`/admin/dsar`（`super_admin` のみ）

---

## このページの目的

メールアドレスを入力し、それに紐づくシステム内のすべての連絡先を検索して、各件の**個人データの範囲**（対話記録・名刺・ニュースレター受信数）を一覧表示します。データ主体からのデータ照会／エクスポート／削除の請求に対応するために使います。

- 連絡先の `email` または `第二 email` を**完全一致**（大文字小文字を区別しない）で照合します。
- **削除済み（ゴミ箱）の連絡先も含みます**——データ主体の請求は、ソフト削除の有無にかかわらず全レコードを対象とする必要があるためです。削除済みには「削除済み」ラベルが付きます。
- そのアドレスの**送信可否の判定**（送信できるか、できない理由）も表示します。判定ロジックは[配信停止・同意管理](suppressions.md)と共通です。
- 各連絡先はそのまま「表示」またはエクスポートできます。

---

## GDPR の背景

DSAR（Data Subject Access Request、データ主体アクセス請求）は、GDPR が個人に認める権利です。本人は、会社が保有する自分の個人データを知る／取得する／削除するよう請求できます。このページでは、super_admin が請求を受けた際に次のことができます：

1. メールをインデックスに、その人に紐づく**すべて**のレコードを検索（ゴミ箱を含む）——アクセス権に対応。
2. データ範囲を数値化し、連絡先ごとに完全な個人データ JSON をエクスポート——アクセス権／データポータビリティ権に対応。
3. そのアドレスの現在の送信可否／抑制状態を確認——異議申立権／削除後の対応に対応。

> 説明責任のため、**照会のたびに監査ログへ記録されます**（アクション `dsar_lookup`、[監査ログ](audit-log.md)では「DSAR データ照会」）。対象には照会したメールが記録されます。

---

## 利用できる人

- 本ページを開けるのは `super_admin` のみです。
- バックエンド API `/api/admin/dsar` は**権限を再検証**します：未ログインは 401、super_admin でなければ 403（`/api/admin/*` は auth middleware の対象外のため、ルート側で自ら保護します）。

---

## 使い方

1. 検索ボックスに**完全な**メールアドレスを入力します（形式が不正な場合は「有効なメールアドレスを入力してください」と表示）。
2. 「検索」を押します。
3. 上部にそのアドレスの送信可否カードが表示され、下部に該当するすべての連絡先とその個人データ範囲が一覧表示されます。
4. 各行で次ができます：
   - **表示** → 連絡先ページ `/contacts/{id}` を開く。
   - **エクスポート** → その連絡先の完全な個人データ JSON をダウンロード（下記参照）。

該当がない場合は「該当する連絡先が見つかりません」、検索前は入力案内が表示されます。

---

## 結果の列

| 列 | 説明 |
|------|------|
| 名前 | 連絡先の名前。ソフト削除済みには「削除済み」ラベル |
| 会社 | 連絡先の会社 |
| 作成者 | この連絡先を作成したユーザー（表示名） |
| 作成日 | 連絡先の作成日 |
| 対話記録 | 連絡先の `interaction_logs` 件数 |
| 名刺 | 連絡先の `contact_cards` 件数 |
| ニュースレター | 連絡先の `newsletter_recipients` 件数 |

---

## 個人データのエクスポート（GDPR データエクスポート）

各行の「エクスポート」は `/api/contacts/{id}/export` を呼び出し、連絡先のすべての個人データを集約した単一の JSON ファイル（`contact-{id}-export.json`）を返します：

- 連絡先本体、名刺、写真、対話記録、タスク、ニュースレター受信者、Email イベント。
- 名刺と写真の**画像本体は Storage に保存**されているため、エクスポートには URL／パスのみが含まれ、画像のバイト列は含まれません。

> エクスポート権限は super_admin または「連絡先のエクスポート」機能付与のいずれかで、super_admin は両方を満たします。エクスポートも[監査ログ](audit-log.md)に記録され（アクション：GDPR データエクスポート）、データ種別ごとの件数が残ります。

---

## 注意事項

- 本ページは super_admin のみ利用でき、バックエンド API でも権限を再検証します。
- これは閲覧専用の照会ページです——ここでデータを編集・削除することはありません。削除は[ゴミ箱](trash.md)または連絡先ページで行います。
- 送信可否のソース一覧と判定方法の詳細は[配信停止・同意管理](suppressions.md)を参照してください。
