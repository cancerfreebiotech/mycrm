# ユーザー管理

パス：`/admin/users`

アクセス権限：`super_admin`、または「ユーザー管理」機能を付与された一般メンバー。付与されたメンバーは **MFA のリセット**と **Telegram ID の編集**のみ可能で、ロール変更・機能権限の調整・メンテナンスモードの切り替えはできません。

---

## ユーザー一覧

システムにログインしたすべてのメンバーを表示します。各カラムのヘッダーをクリックして並べ替えできます：

| カラム | 説明 |
|--------|------|
| 氏名 | ユーザーが設定した表示名 |
| Email | Microsoft アカウント（主キー） |
| Telegram | 紐付けられた Telegram ID。その場で編集可能 |
| Teams | Microsoft Teams が連携済みか |
| ロール | member / super_admin |
| 最終ログイン | 直近のログイン日時 |
| MFA | 多要素認証を設定済みか |

モバイルでは同じ情報がカード形式で表示されます。

---

## ロール管理

> super_admin のみ操作可能。

該当メンバーの「スーパー管理者に昇格」／「一般メンバーに降格」ボタンをクリックしてロールを切り替えます。

- `member` → 一般ユーザー
- `super_admin` → すべての管理機能にアクセス可能

> 自分のロールは変更できません。また、すべての管理者が降格されないよう、少なくとも 1 名の super_admin を維持してください。

---

## 機能権限の付与

> super_admin のみ操作可能。member アカウントにのみ表示されます。

メンバーの行には機能権限のタグ（chips）が並び、クリックで付与・取り消しができます。付与済みの項目には ✓ が表示されます。付与可能な機能：タグ、メールテンプレート、プロンプト、国、ニュースレター、スキャン失敗審査、重複連絡先、名刺インポート、ゴミ箱、ユーザー管理、および未割り当てメモ・連絡先エクスポート・一括メールなど。

付与されたメンバーには、対応する項目がサイドバーに表示されます。

---

## MFA 状態とリセット

MFA カラムは、メンバーが多要素認証を完了しているか（設定済み／未設定）を表示します。

いずれかの認証器を設定済みのメンバーの行には「リセット」ボタンが表示されます。クリックすると確認が必要で、リセットするとそのメンバーの既存の MFA 認証器がすべて削除され、メンバーは**次回ログイン時に MFA を再設定する必要があります**。

この操作は super_admin と「ユーザー管理」を付与されたメンバーの両方が実行できます。

---

## Telegram ID の編集

Telegram カラムの ✏️ をクリックしてインライン編集し、数字を入力して保存します：

- 正の整数である必要があり、そうでない場合は拒否されます。
- 9 桁未満（一般的な Telegram ID は 9〜10 桁）の場合は警告が表示され、確認後に保存されます。
- 空欄にして保存すると連携が解除されます。

この操作は super_admin と「ユーザー管理」を付与されたメンバーの両方が実行できます。

---

## Teams 連携

Teams カラムは連携状態（連携済み／未設定）のみを表示する読み取り専用です。連携は、メンバーが Teams でアカウントをリンクした際に作成されます。

---

## メンテナンスモード

> super_admin のみ操作可能。

ページ上部にメンテナンスモードの切り替えがあります。有効にすると一般ユーザーはメンテナンスページにリダイレクトされ、super_admin のみがシステムを引き続き利用できます。有効化には確認が必要です。

---

## 注意事項

- ユーザーアカウントは**初回ログイン時に自動作成**されます。手動で事前作成することはできません
- `@cancerfree.io` ドメインの Microsoft アカウントのみログインできます
- ユーザーの削除：現時点では Supabase Dashboard から DB を直接操作する必要があります
