# MCP トークン

パス：`/admin/mcp-tokens`（`super_admin` のみ）

---

## このページの目的

外部 AI エージェントが（MCP 経由で）本システムにアクセスする際に使うアクセスキーを管理します。発行済みトークンをテーブル一覧し、列は：名称／用途、割り当て先、Scopes、最終使用、有効期限、状態（有効／無効／期限切れ）。右上に「MCP アクティビティ」へのリンクがあります。

---

## トークンの発行

「トークンを発行」でダイアログを開き、入力します：

- **名称**（用途）と**説明**。
- **割り当て先**：ユーザーを 1 人選択。エージェントの操作はそのユーザーとして記録されます。
- **Scopes**：許可する権限を選択。読み取り系（contacts / newsletter / tags）と書き込み系（contacts / notes / newsletter）。ある scope が割り当て先のシステム内機能権限を超える場合は警告が出て、確認しないと続行できません。
- **任意の actor を許可**：任意。
- **有効期限**：無期限 / 1 年 / 30 日 / 24 時間。

発行後、**平文キーは一度だけ表示**されます。すぐにコピーして保管してください。閉じると再表示できません。

---

## 既存トークンの管理

各行で：そのトークンのアクティビティログへ移動、無効化／有効化、削除（いずれも確認が必要）。無効または期限切れのトークンは使用できません。

---

## 注意事項

- 平文キーは発行時に一度だけ表示されます。紛失した場合は削除して再発行するしかありません。
- 付与する scope は割り当て先のシステム内権限と一致させるべきです。権限を超える scope にはシステムが警告します。
