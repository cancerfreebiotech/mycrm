# システム健全性

パス：`/admin/health`（`super_admin` のみ。サイドバーは super_admin にのみ表示され、データ API もバックエンドで権限を検証します）

---

## このページの目的

システム全体の稼働状況を 1 ページで俯瞰します。外部サービスの健全性、Email 検索ツール、そして v7.4.0 で追加された**スケジュール心拍・デッドレター・今月の使用量**の 3 区画を含みます。

上部で「今すぐチェック」でき、「30 秒ごとに自動更新」にチェックすると継続監視できます。最終チェック時刻も表示されます。

---

## 外部サービス状態

サマリーバーが正常／異常／未設定のサービス数を集計し、下のカードで各サービス（Supabase、Gemini、Telegram Bot、SendGrid、Teams Bot）を一覧します。

- 状態バッジ：**正常** / **異常** / **未設定**。
- 各カードにレイテンシーバー：緑（< 500 ms）、黄（< 2000 ms）、赤（それ以上）。下部に色の凡例があります。

---

## Hunter.io（Email 検索）

「email がない」連絡先について、Hunter.io で相手の email を探します。

- **API Key**：キーを貼り付けて保存。設定済みなら緑のチェックが表示。
- **統計**：email なし総数、未検索、検索したが未発見、今月検索済み。残クレジット情報があれば今月分を表示。
- **検索開始**：未検索の連絡先に対して照会し、終了後に発見／未発見のリストを表示。先に API Key の設定が必要。
- **検索状態のリセット**：検索フラグを消去し、連絡先を再度検索可能にします（確認が必要）。
- **モジュール無効の通知（v7.9.6）**：「Hunter.io 自動補完」が[組織設定](org-settings.md)でオフの場合、検索を実行すると「モジュール無効」の通知と組織設定へのリンクが表示され、実際の照会やクレジット消費は行われません。

---

## スケジュール心拍（v7.4.0）

各バックグラウンドジョブ（cron）の直近の実行状況を一覧します：

- 状態：**正常** / **遅延** / **エラー** / **記録なし**。遅延やエラーがあると区画全体が警告色になります。
- 最終完了時刻（相対時間）と実行所要時間を表示。

夜間自動化やレポートチェックなどのスケジュールが実際に動いているかを素早く確認できます。

---

## デッドレター（v7.4.0）

各テーブルで処理に失敗した件数（デッドレター）を一覧します：

- 失敗件数のあるテーブルは赤で示され、展開すると直近のエラーメッセージを確認できます。
- `pending_contacts`、`contact_briefings` には「再キュー投入」ボタンがあり、失敗項目をキューへ戻せます。
- `failed_scans` には「スキャン失敗審査」へのリンクがあります。

---

## 今月の使用量と予算しきい値（v7.4.0；しきい値編集は v7.9.5）

今月の各使用量を先月と対比してカード表示します：AI 呼び出し回数、AI 入力／出力トークン（記録がある場合）、送信 Email 数、ニュースレター送信数。

v7.9.5 以降、各カードで「月間上限」を直接入力し「しきい値を保存」で保存できます：

- 空欄＝**上限なし**。カードは先月との比較を表示します。
- 上限を設定すると、カードは使用量の進捗バーと割合を表示します：**緑 < 80%**、**黄 80–99%**、**赤 ≥ 100%**。
- 80% / 100% 到達時に、既存の監視を通じて組織オーナーへ自動通知されます（通知自体は既存機能）。

---

## 注意事項

- 本ページは super_admin のみ。各区画はそれぞれの API から取得するため、1 つが失敗してもその区画のみエラー表示され単独で再試行でき、他区画には影響しません。
- Hunter.io の無料枠は限られています。検索前に残クレジットを確認してください。
